# StockPulse 交易感知系统 — AI Agent 驱动构建成果

**使用的 AI/Agent 工具**：Claude Code（CLI 端到端编码）、Anthropic API（多角色分析推理）、FastMCP（微信抓取工具暴露）

## 1. 项目解决的核心痛点

散户投资者每天面对大量微信股票推荐文章，人工阅读比对耗时巨大且易遗漏关键分歧。行情数据碎片化分布在多个市场（A 股/港股/美股/韩日/加密/期货），缺少统一的实时监控与 AI 辅助决策终端。

## 2. 核心逻辑流

**采集 → 分析 → 信号 → 驾驶舱 → 验证闭环**

- **采集层**：Playwright 抓取微信公众号文章，通过 FastMCP 协议暴露为 AI 可直接调用的 Tool
- **分析层**：AI 把同一批博主文章综合成一份结构化报告（市场总览 / 热门板块 / 操作建议 / 观点分歧 / 风险提示），并自动抽取提及的股票入库跟踪；个股做五维情绪雷达，财报做四维度解读
- **信号层**：市场情绪聚合 + 热门股排名 + 行业涨跌 + 6 大指数分时图 5 秒级刷新
- **决策层**：K 线三数据源实时竞速取最快响应 + 形态规则引擎 + 自然语言选股
- **验证闭环**：模拟交易 → 交易日志 → 胜率统计

## 3. 量化成果

| 维度 | 数据 |
|------|------|
| **代码规模** | 46 个 Python 源文件，~9,000 行后端代码 + 6,748 行前端（单文件 SPA） |
| **提交历史** | 132 次 Git 提交，30 天内连续迭代 |
| **API 端点** | 94 个路由端点，10 个路由模块 |
| **并发能力** | 180 个异步函数，全链路 async/await |
| **数据源覆盖** | 6 类市场（A 股/港股/美股/韩日/期货/加密 Top50），K 线三源竞速（AKShare + 腾讯 + mootdx TCP ~50ms） |
| **缓存策略** | SQLite 持久化，6 级 TTL 梯度（5s → 30s → 5min → 30min → 1h → 24h） |
| **技术指标** | 9 个纯 NumPy 实现（RSI、MACD、KDJ、Bollinger、均线等） |
| **AI 分析** | 多博主文章综合成结构化报告 + 自动抽取提及股票；个股五维情绪雷达；财报四维度解读 |
| **数据库** | SQLite WAL 模式，9 张业务表，增量迁移 |
| **前端页面** | 单页应用 9 个视图（行情/驾驶舱/技术分析/北向资金/龙虎榜/自选/模拟交易/日志/统计） |
| **文档沉淀** | 42+ 篇调研/计划/设计文档，精简整合为 4 份核心文档 |
| **实时性能** | 驾驶舱 5s 自动刷新 6 指数分时，K 线三源竞速取最快响应（mootdx TCP 直连 ~50ms） |

## 4. AI 分析能力

| 能力 | 说明 |
|-----------|---------|
| 博主文章综合分析 | 多篇文章 → 市场总览 / 热门板块 / 操作建议 / 观点分歧 / 风险提示 的结构化报告 |
| 提及股票抽取 | `extract_mentions` 自动从文章抽出股票,入库跟踪 |
| 个股情绪雷达 | 价量 / 资金 / 热度 / 博主 / 舆情 五维评分 + AI 一句话判读 |
| 财报四维解读 | 成长性 / 盈利质量 / 财务健康 / 风险，A 股 + 美股 |
| 行业六步分析 | 行业边界 → 生命周期 → 阶段聚焦 → 估值 → PEST → 景气度跟踪 |
