# 15 — 板块页面优化设计

**日期**：2026-05-11  
**状态**：待实现  
**涉及文件**：`sector_service.py`、`routers/sector.py`、`templates/index.html`

---

## 背景与问题

板块功能上线后发现三个明显缺陷：

1. **成分股无法显示** — 点击板块详情后，成分股区域提示"数据不可用"。根因是 `_normalize_stocks()` 未提取东方财富接口返回的完整字段（PE、市值、换手率等），以及缺乏足够的错误日志定位板块名称匹配问题。
2. **K 线历史不全** — `_fetch_kline_eastmoney` 未传 `start_date`，导致接口返回数据量不定；前端默认 `count=120` 约 6 个月日K，无法满足长周期分析需求。
3. **涨跌热力图视觉较差** — 现有 flex-wrap + 按成交额动态尺寸的方案，块大小不一、排列混乱，信息密度低。

---

## 设计方案

### 1. 成分股修复与字段扩充

**目标**：修复显示异常，并展示丰富字段（代码、名称、最新价、涨跌幅、PE动、流通市值、总市值、换手率、今日最高、今日最低）。

**后端（`sector_service.py`）**

`_normalize_stocks()` 扩充提取以下列（东方财富 `stock_board_industry_cons_em` 已返回，只需补充映射）：

| 字段 key | AKShare 列名 |
|---------|------------|
| `pe` | `市盈率动` |
| `circ_mv` | `流通市值` |
| `total_mv` | `总市值` |
| `turnover_rate` | `换手率` |
| `high` | `最高` |
| `low` | `最低` |

同时在 `get_board_stocks` 的 `except` 块里改为 `logger.exception()`（输出完整堆栈），方便排查板块名称不匹配问题。

**前端（`index.html` — `loadSectorBoardStocks`）**

成分股表格列扩充为：

| 代码 | 名称 | 最新价 | 涨跌幅 | PE(动) | 流通市值 | 换手率 |
|-----|-----|--------|--------|--------|---------|--------|

- 市值格式化：≥1亿显示"X.X亿"，≥1万亿显示"X.X万亿"
- PE 异常值（≤0 或 >500）显示为 `—`
- 点击行仍跳转股票详情

---

### 2. K 线历史数据修复

**目标**：日K 看 2 年（约 500 交易日），周K/月K 全量获取。

**后端（`sector_service.py` — `_fetch_kline_eastmoney`）**

根据 `period` 参数自动计算 `start_date`：

```
daily   → today - 730 天
weekly  → today - 3650 天（10 年）
monthly → today - 3650 天（10 年）
```

调用方式改为：
```python
ak.stock_board_industry_hist_em(
    symbol=board_name,
    period=ak_period,
    start_date=start_date,   # 新增
    end_date=end_date,       # 新增（今日）
    adjust=""
)
```

`get_board_kline` 中 `count` 参数保留，作为上限截断（默认改为 500）；若 count=0 则不截断。

**前端（`index.html`）**

- `loadSectorKline` 默认请求 `count=500`
- 周K / 月K 请求 `count=0`（全量）

### 3. 热力图重设计 — 均匀网格（方案 B）

**目标**：等大方格，按涨跌幅从高到低排列，颜色深浅表示幅度大小，视觉清晰。

**布局规则**

```css
display: grid;
grid-template-columns: repeat(auto-fill, minmax(88px, 1fr));
gap: 5px;
```

每个方格固定高度 `64px`，内容居中：上行板块名（12px bold），下行涨跌幅（14px bolder）。

**颜色方案**（与现有全局配色保持一致）

| 场景 | 背景色 | 文字色 |
|------|--------|--------|
| 涨幅 ≥ 3% | `rgba(239,68,68,0.90)` | `#fff` |
| 涨幅 1~3% | `rgba(239,68,68,0.55)` | `#dc2626` |
| 涨幅 0~1% | `rgba(239,68,68,0.25)` | `#dc2626` |
| 平盘 | `rgba(148,163,184,0.3)` | `#64748b` |
| 跌幅 0~1% | `rgba(34,197,94,0.25)` | `#16a34a` |
| 跌幅 1~3% | `rgba(34,197,94,0.55)` | `#16a34a` |
| 跌幅 ≥ 3% | `rgba(34,197,94,0.90)` | `#fff` |

**交互**：hover 时 `transform: scale(1.04)` + `box-shadow`；点击进入板块详情（现有逻辑不变）。

**排序**：渲染前先按 `change_pct` 降序排列（涨幅最高的在左上角）。
