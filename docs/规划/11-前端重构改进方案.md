## 二、问题清单

### 2.1 CRITICAL — 安全漏洞

#### XSS via innerHTML（17+ 处注入点）

所有通过模板字符串拼接 HTML 的地方，如果数据包含 `<script>` 或事件处理器就会触发 XSS。

| # | 函数 | 行号 | 注入数据 | 风险 |
|---|------|------|---------|------|
| 1 | `showToast()` | 2010 | `message` 参数 | 低（内部调用为主） |
| 2 | `renderBloggerManageList()` | 2234-2251 | `b.name`, `b.article_title` | **高**（博主编了恶意内容） |
| 3 | `refreshBloggerCheckList()` | 2277-2301 | `b.name`, `b.article_title` | **高** |
| 4 | WebSocket 消息处理 | 2508 | `msg.message` | 中 |
| 5 | QR 登录 | 2712, 2718 | `msg.image`, `msg.token` | 中 |
| 6 | `searchStock()` | 2947-2956 | `s.code`, `s.name`, `s.market` | **高**（外部数据源） |
| 7 | `refreshQuote()` | 3035-3047 | `d['最新价']` 等行情字段 | 中 |
| 8 | `loadShareholders()` | 3335-3349 | `h.name`, `h.type` | 中 |
| 9 | `loadAnnouncements()` | 3361-3365 | `n.title`, `n.url` | **高**（URL 可为 `javascript:` 协议） |
| 10 | `loadStockNews()` | 3375-3382 | `n.title`, `n.summary`, `n.url` | **高** |
| 11 | `_renderIndustryGrid()` | 3585-3591 | `name` 拼入 onclick | **高**（仅转义单引号） |
| 12 | `loadWatchlist()` | 4364-4377 | `w.symbol`, `w.note` | **高**（用户可编辑内容） |
| 13 | `loadSim()` 持仓 | 4453-4468 | `p.symbol`, `p.note` | **高** |
| 14 | `loadSim()` 历史 | 4480-4491 | `h.symbol`, `h.note` | **高** |

**修复方案**：引入 `escapeHtml()` 工具函数，所有 innerHTML 注入点统一转义。对 URL 字段增加 `javascript:` 协议过滤。

```javascript
// 需新增的工具函数
function escapeHtml(str) {
  if (str == null) return '';
  return String(str)
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#39;');
}

function safeUrl(url) {
  if (!url) return '#';
  const u = String(url).trim().toLowerCase();
  if (u.startsWith('javascript:') || u.startsWith('data:') || u.startsWith('vbscript:')) return '#';
  return url;
}
```

## 四、技术选型建议

### 保持原生 JS 的理由

- 个人项目，单人开发，无需 React/Vue 的组件复用和团队协作能力
- 零构建部署简化运维
- 当前 4544 行用原生 JS 完全可以整理好，问题不在语言而在工程纪律

### 如果未来要迁移框架

推荐 **Vite + 原生 JS（无 React/Vue）**：
- Vite 提供模块化、HMR、tree-shaking
- 无需学习新框架，现有 JS 代码几乎原样迁移
- 构建产物天然压缩、代码分割
- 比 React 迁移成本低 80%，但获得 80% 的工程化收益
