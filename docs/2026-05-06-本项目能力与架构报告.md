## TL;DR（电梯陈述）

**这是一个 5000 行 Python 实现的「多市场行情 + 微信博主聚合 + 多视角 AI 投资分析」混合型平台**，同时提供 Web 应用和 MCP 服务两种形态。差异化亮点是**三视角并行 AI 分析（价值派/成长派/趋势派）**、**自定义 MA 周期（1-240）**、**财务规则引擎**和**MCP 一等公民**。当前阶段：**功能完整度高、产品创意鲜明，但工程化程度不足（零测试、JSON 存储、单文件路由 750 行）**——适合作为求职作品集和个人决策工具，距商业化产品还有一段工程鸿沟。

## 1. 项目定位

### 1.1 一句话定位

**"一个集多市场行情查询、微信博主文章聚合、多视角 AI 投资分析、以及 MCP 协议接入于一体的股票信息服务平台"**

更准确地说，这是一个**混合型投资信息决策助手**：

- **主体**：Web 应用（FastAPI + WebSocket），面向终端用户提供博主文章分析、股票查询、基金期货行情
- **附加**：MCP Server，供 Claude 等 AI 客户端调用微信文章读取能力
- **核心差异化**：多投资视角（价值派 / 成长派 / 趋势派）的 AI 分析，而非传统单一观点汇总

### 1.2 目标用户画像

| 用户群 | 痛点 | 本项目方案 |
|---|---|---|
| 中文资深散户 | 微信公众号信息散乱、缺少结构化 | 博主聚合 + 自动抓取 + AI 分析 |
| 多市场配置者 | A/H/U/期货/基金分散在 N 个 App | 一界面切换全市场 |
| AI 重度使用者 | 通用 AI 无金融语境 | MCP 工具 + 三视角 Prompt |
| 求职 PM / 开发者 | 需要可展示的金融产品作品集 | 这就是作品集本身 |

## 2. 完整能力清单

### 2.1 数据采集

#### 微信文章采集
- **文件**：`src/scraper.py`（163 行）+ `src/parser.py`（75 行）
- **方案**：Playwright 浏览器自动化 + 反检测（隐藏 webdriver 标志、自定义 UA）
- **可靠性**：3 次重试 + SIGPIPE / 网络超时检测
- **输出字段**：title / author / publish_time / content / cover_url
- **入口**：`app.py:728-815`（`_scrape_urls`）

#### 博主信息识别
- **文件**：`src/blogger.py`（560 行）
- **方案**：从公众号文章 URL 反向识别博主（biz / user_name / avatar / name）
- **解析优先级**：htmlDecode → var = → URL 参数提取
- **入口**：`blogger.py:54-94`（`_extract_blogger_info`）

#### A 股市场数据
- **文件**：`src/stock_service.py`（420 行）
- **数据矩阵**：

| 实时行情 | 腾讯股票 API（qt.gtimg.cn）| 16 项：实时价/涨跌幅/成交量/市值/换手率/振幅等 |
| K 线 | 腾讯前复权日线 API | 支持日/周/月线，MA 1-240 |
| 公司信息 | AKShare 巨潮 | `stock_profile_cninfo` |
| 财务数据 | AKShare 同花顺 | `stock_financial_abstract_ths`，10+ 指标 |
| 资金流向 | AKShare 东方财富 | `stock_individual_fund_flow` |
| 个股新闻 | AKShare | `stock_news_em` |
| 公告 | 巨潮资讯接口 | 新增于 commit `8d9188e` |
| 股东信息 | AKShare | - |

- **市场识别**：`stock_utils.py:63-94`（A/沪/深/创/科创/北交所）

#### 全球市场（韩 / 日股）

### 2.2 数据处理与分析

#### K 线处理
- 日 → 周 / 月聚合：`src/stock_service.py:115-131`
- MA 自定义周期 1-240（commit `ce7f105`）
- 前复权处理
- 路由：`app.py:207-212`（`/api/stock/kline/{symbol}`）

#### 财务指标高亮（规则引擎）
- **配置**：`src/financial_rules.json`（44 行）
- **规则**：净利润同比 / 资产负债率 / ROE / 毛利率 / EPS / 净利润
- **颜色**：red（异常）/ green（优秀）/ orange（警示）
- **示例**：ROE > 25% → ★ 绿色；负债率 > 70% → ▲ 红色

#### AI 投资分析（核心差异化）
- **文件**：`src/analyzer.py`（200+ 行）+ `src/agents/personas.py`（127 行）
- **三大投资视角**：

| 视角 | 关注点 | Prompt 关键词 |
|---|---|---|
| **价值派**（buffett） | 护城河、ROE、自由现金流、估值 | PE / PB / 股息率 |
| **成长派**（growth_lynch） | 营收/利润增速、PEG、行业渗透率 | 产品迭代、市场空间 |
| **趋势派**（trend_follower） | 板块轮动、量价配合、强势股梯队 | 技术信号 |

- **并行分析**：多视角同时跑（`analyzer.py:80-113`）
- **AI 接入**：OpenAI / Claude 兼容 API（`AI_API_KEY` / `AI_BASE_URL` / `AI_MODEL`）
- **降级**：无 AI 时自动文本汇总（`_fallback_report`）

#### 报告生成
- 内容：市场总览 / 热门板块 / 操作建议 / 风险提示 / 观点分歧 / 今日要点
- 格式：Markdown，可邮件 / HTML 导出
- 入口：`app.py:927-966`

### 2.3 服务形态

#### Web 应用（FastAPI）
- **框架**：FastAPI 0.104+ / Uvicorn 0.24+ / Pydantic
- **前端**：纯 HTML 单文件（`templates/index.html`，4129 行 / 19.6KB）
- **图表**：LightweightCharts.js（K 线、面积图）
- **设计系统**：CSS 变量（color / spacing / radius / shadow / motion tokens）
- **布局**：Flexbox 侧栏（228px）+ 主内容区
- **实时**：WebSocket + JSON 协议
- **端口**：8000

#### MCP 服务器
- **文件**：`src/server.py`（70 行）
- **框架**：fastmcp
- **暴露工具**：`read_weixin_article(url: str)`
- **客户端**：Claude Desktop 等

#### 实时任务管道
- **WebSocket**：`/ws/task`（`app.py:879-1019`）
- **三阶段**：
  1. 文章抓取（并发，每篇间隔 2s 可配）
  2. 报告生成（AI 或文本汇总）
  3. 邮件投递（可选）
- **可取消**：客户端 `cancel` 信号

#### 扫码登录
- **WebSocket**：`/ws/mp-login`（`app.py:502-723`）
- **能力**：自动化提取微信公众号后台 Cookie + Token
- **流程**：浏览器启动 → 二维码 → 轮询 → 提取 cookies → 自动保存
- **状态**：扫码 / 失效 / 成功多种推送

### 2.4 用户交互（5 大 Tab）

| Tab | 视图 | 关键交互 |
|---|---|---|
| **博主管理** | 列表 / 添加 / 删除 / 批量刷新 | Cookie 测试、扫码登录 |
| **文章分析** | URL 直输 / 选博主 / 视角多选 | 实时进度条、AI 预览、邮件发送 |
| **股票查询** | 多市场搜索 / 5 关键指标卡片 / 详情弹窗 | K 线日 / 周 / 月 + MA 自定义、财务规则高亮 |
| **板块扫描** | 市场切换器（A/U/H/币/基金/期货）| 板块热力、成分股、持仓弹窗、龙虎榜 |
| **配置** | SMTP / AI API / Cookie | 有效性测试 |

### 2.5 自动化与缓存

### 4.2 模块边界评估

**优秀解耦**

- ✅ 市场提供者抽象（`market/__init__.py` 注册表模式）—— 易扩展新市场
- ✅ 配置 / 业务分离（`config.py`）
- ✅ 缓存 / 业务分离（`stock_utils.TTLCache`）
- ✅ 爬虫 / 解析分离（`scraper.py` vs `parser.py`）

**需要改进的耦合**

- 🔴 **app.py 750 行职责过重**：路由 + WebSocket + 配置 + 扫码 + 任务编排混合
- 🟡 **blogger.py 560 行**：CRUD（文件 I/O）+ 爬虫 + HTML 解析三件事混合
- 🟡 **全局状态**：`scraper` / `config` / `blogger_mgr` 在 app.py 全局初始化，难测试
- 🟡 **index.html 4129 行**：HTML / CSS / JS / 路由全混

### 4.3 代码质量

**强项**
1. 异步完整性：大多数 I/O 都是 async
2. 错误处理：爬虫 3 次重试，关键路径 try-except
3. 类型提示：Pydantic models（`ConfigRequest` / `IdentifyRequest`）
4. 安全意识：敏感信息环境变量 + 自动脱敏
5. 可配置性：SMTP / AI / 微信凭证均环境变量

**弱项**
1. 🔴 **零测试**：无 pytest 单元测试、无集成测试
2. 🔴 **缺 .gitignore**：`user_config.json` 风险被提交
3. 🟡 **错误消息通用**：缺少失败诊断细节
4. 🟡 **无 OpenAPI 文档**：前端硬编码 URL
5. 🟡 **依赖未锁版本**：requirements.txt 未 freeze
