# 驾驶舱（行情总览仪表盘）设计文档

## 目标

在 StockPulse 侧边栏新增「驾驶舱」页面，作为默认首页，提供 A 股整体行情一屏总览：
- 上半部分：市场情绪面板（4 张卡片）
- 下半部分：6 个核心指数的实时分时图

## 后端 API

新增路由模块 `routers/cockpit.py`，注册 3 个端点。

### 1. GET /api/cockpit/sentiment

返回市场情绪聚合数据。缓存 TTL 15s。

**数据源：** AKShare `ak.stock_changes_em`（涨跌家数）、`ak.stock_zt_pool_em` / `ak.stock_zt_pool_dt_em`（涨跌停）、`ak.stock_market_activity_legu`（资金流向）

**响应结构：**
```json
{
  "rise_fall": { "up": 2100, "down": 2500, "flat": 300, "ratio": 0.84 },
  "limit": { "up_limit": 45, "down_limit": 12, "broken": 8, "consecutive_2": 15 },
  "volume": { "total_yuan": 1200000000000, "yesterday_yuan": 1100000000000, "change_pct": 9.1 },
  "flow": { "main_net": -5000000000, "super_large_net": -2000000000, "large_net": -1500000000, "mid_net": -1000000000, "small_net": 9500000000 }
}
```

### 2. GET /api/cockpit/indices

返回 6 个指数的实时报价快照。缓存 TTL 5s。

**目标指数：** 上证指数、深证成指、创业板指、科创50、沪深300、中证1000

**数据源：** 腾讯行情接口 `qt.gtimg.cn`，复用 `stock_service.py` 中已有的腾讯 API 调用逻辑。

**响应结构：**
```json
[
  {
    "code": "sh000001",
    "name": "上证指数",
    "price": 3251.23,
    "change_pct": 0.34,
    "change": 11.05,
    "volume": 1234567,
    "amount": 45678901234,
    "prev_close": 3240.18
  }
]
```

### 3. GET /api/cockpit/tick/{code}

返回单个指数当日的分时数据（1 分钟线）。缓存 TTL 5s。

**数据源：** AKShare `ak.index_zh_a_min(symbol=code, period="1", adjust="")`

**URL 参数：** `code` — 指数代码，如 `sh000001`

**响应结构：**
```json
{
  "code": "sh000001",
  "name": "上证指数",
  "prev_close": 3240.18,
  "data": [
    { "time": "09:31", "price": 3241.23, "volume": 1234567 },
    { "time": "09:32", "price": 3242.56, "volume": 2345678 }
  ]
}
```

### 情绪面板卡片

| 卡片 | 内容 | 视觉元素 |
|------|------|---------|
| 涨跌家数 | 上涨/下跌/平盘家数 + 涨跌比 | 红绿渐变进度条 |
| 涨跌停 | 涨停/跌停/炸板/连板家数 | 大字体数字 + 标签 |
| 成交量 | 两市成交额 + 较昨日变化 | 增减箭头 + 百分比 |
| 资金流向 | 主力净流入 + 大/中/小单 | 红绿色柱状图 |

### 指数分时图卡片

- 卡片头：指数名称 + 最新价 + 涨跌幅
- 卡片体：Canvas 渲染的面积分时图
- 涨为红色、跌为绿色，背景色微调呼应
- 显示昨日收盘价水平虚线作参考基准

## 数据流与自动刷新

### 页面加载流程

1. 用户切换到「驾驶舱」→ `switchView('cockpit')`
2. `initCockpit()` 并行请求：
   - `fetchSentiment()` → 渲染情绪面板
   - `fetchIndices()` → 渲染 6 个指数报价
   - 6 个 `fetchTick(code)` → 渲染 6 个分时图（首屏优先加载前 2 个，其余异步）
3. 启动 5s 定时器持续刷新

### 自动刷新策略

- 5s `setInterval` 同时刷新情绪数据和指数报价
- 分时图增量追加：每次只请求最近 N 分钟数据，追加到已有数据后，避免全量重绘
- 离开页面时 `clearInterval`（复用 `switchView` 中的 `clearTimers()`）

### 错误处理

- 单个 API 失败不影响其他卡片，显示"加载失败"占位 + 重试按钮
- 非交易时段涨跌停数据无数据时，显示最近交易日数据 + "休市"标记
- 网络超时（>5s）跳过本轮刷新，不阻塞下一轮

### UI 状态

- 首次加载：骨架屏 shimmer 动画
- 刷新中：卡片右上角灰色小圆点闪烁
- 非交易时段：顶部横幅"非交易时段，数据为最近收盘快照"
