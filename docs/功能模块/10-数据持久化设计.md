## 1. 总体方案

- **选型**：SQLite，Python 标准库 `sqlite3`，零新依赖
- **数据库文件**：`wexin-read-mcp-main/data.db`，项目根目录，加入 `.gitignore`
- **原则**：不引入 ORM，手写 SQL；WAL 模式；单连接管理

**跨电脑同步策略**：

| 数据 | 方案 | 同步方式 |
|---|---|---|
| bloggers | **保持 `bloggers.json`** | git push/pull，天然同步 |
| 行情/财务缓存 | SQLite `cache` 表 | 不需要同步（每台电脑各自重建） |
| watchlist | SQLite `watchlist` 表 | 后续加 JSON 导出/导入功能 |
| 投资组合 | SQLite `portfolios` + `positions` | 后续加 JSON 导出/导入功能 |
| 回测结果 | SQLite `backtests` 表 | 后续加 JSON 导出/导入功能 |

**保留不变的**：`user_config.json`（配置）、`financial_rules.json`（静态规则）、精选股票列表（代码常量）、Persona（dataclass）

**`data.db` 加入 `.gitignore`**——每台电脑独立，不进入版本控制。

## 2. 数据库 Schema

### 2.1 cache（替代内存 TTLCache）

```sql
CREATE TABLE IF NOT EXISTS cache (
    key         TEXT PRIMARY KEY,
    value       TEXT NOT NULL,     -- JSON 序列化
    expires_at  REAL NOT NULL      -- Unix 时间戳（秒）
);
```

缓存键保持现有格式不变（如 `quote:000001`、`kline:000001:day:250`），所有服务模块无感知切换。

清理策略：
- 读取时检查 `expires_at`，过期视为未命中，删除旧行
- 启动时清理全部过期行：`DELETE FROM cache WHERE expires_at < unixepoch()`
- 不设定时清理任务（SQLite 单写，后台清理反而拖慢）

### 2.2 watchlist（Phase 1 新增）

```sql
CREATE TABLE IF NOT EXISTS watchlist (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    symbol      TEXT NOT NULL,
    market      TEXT NOT NULL,     -- 'A'|'HK'|'US'|'KR'|'JP'|'FUND'|'CRYPTO'|'FUTURE'
    name        TEXT,              -- 展示名称，冗余存储方便离线显示
    added_at    TEXT NOT NULL DEFAULT (datetime('now')),
    note        TEXT DEFAULT '',
    sort_order  INTEGER DEFAULT 0,
    UNIQUE(symbol, market)
);
```

跨电脑同步：后续提供 JSON 导出/导入按钮，手动在电脑间迁移。

### 2.3 portfolios + positions（Phase 2 预留）

```sql
CREATE TABLE IF NOT EXISTS portfolios (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT NOT NULL,
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at  TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE TABLE IF NOT EXISTS positions (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    portfolio_id  INTEGER NOT NULL REFERENCES portfolios(id),
    symbol        TEXT NOT NULL,
    market        TEXT NOT NULL,
    name          TEXT,
    shares        REAL NOT NULL,
    buy_price     REAL NOT NULL,
    buy_date      TEXT NOT NULL,
    created_at    TEXT NOT NULL DEFAULT (datetime('now')),
    UNIQUE(portfolio_id, symbol, market)
);
```

### 2.4 backtests（Phase 3 预留）

```sql
CREATE TABLE IF NOT EXISTS backtests (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT,
    symbol      TEXT NOT NULL,
    market      TEXT NOT NULL,
    strategy    TEXT NOT NULL,
    params      TEXT NOT NULL,     -- JSON
    start_date  TEXT NOT NULL,
    end_date    TEXT NOT NULL,
```

## 3. 模块设计

### 3.1 新增文件：`src/database.py`

提供统一的数据库连接和基础操作，单一职责——只管理连接和建表。

```python
# 对外接口
def get_db() -> sqlite3.Connection   # 获取单例连接
def init_db(db_path=None) -> None    # 建表 + 开启 WAL + 清理过期缓存
```

**连接策略**：
- 单连接（`check_same_thread=False`），所有操作都在 FastAPI 主线程
- 连接在 `init_db()` 时创建，进程生命周期内复用
- 不使用连接池（SQLite 单写，连接池无意义）

### 3.2 改造文件：`src/stock_utils.py`

**删除**：`TTLCache` 类及模块级 `cache` 单例

**新增**：两个纯函数


所有调用方改动极小——`cache.get(key)` 改为 `cache_get(key)`，`cache.set(key, val, ttl)` 改为 `cache_set(key, val, ttl)`。

`bloggers.json` 读写逻辑保持原样。git 管理，天然跨电脑同步。

## 4. 启动流程

```
FastAPI 启动 → init_db() → 建表（如不存在）+ 清理过期缓存 → 正常服务
```

不需要迁移脚本——`cache` 表数据本来就从外部 API 重建，`watchlist` 等新表为空即可。

## 5. 风险与回退

| 风险 | 缓解 |
|---|---|
| `data.db` 被误删 | 数据库由外部 API 数据重建，无永久数据丢失（watchlist 等后续加导出保底） |
| 缓存膨胀 | 启动时清理过期行；估算上限取决于行情数据量（< 50MB） |
| 跨电脑 watchlist 不同步 | Phase 1 后加 JSON 导出/导入功能 |
