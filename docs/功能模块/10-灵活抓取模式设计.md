# 灵活爬取模式 & 问财增强模块设计

## 一、爬取模式设计

### 需求

当前 `fetch_recent_articles()` 只支持"获取最新 N 篇"，每次任务对所选博主一视同仁。实际使用中需求多变：

- 有时只看每个博主最新 1 篇
- 有时需要最近 N 篇深度分析
- 有时只要某时间段内的（如"最近一周的策略观点"）

### 三种模式

| 模式 | 标识 | 参数 | 说明 |
|------|------|------|------|
| 最新一篇 | `latest` | 无 | 每个博主取最新 1 篇文章 |
| 最新 N 篇 | `latest_n` | `count` (默认 5) | 每个博主取最近 N 篇 |
| 时间段筛选 | `period` | `period` (预设值) | 只取该时间段内的文章 |

时间段预设值：`today`、`last_3_days`、`last_week`、`last_month`

### 后端改动

**文件**: `src/blogger.py`

`fetch_recent_articles()` 签名变更：

```python
async def fetch_recent_articles(
    self, blogger: dict, count: int = 5, mode: str = "latest_n", period: str | None = None
) -> dict:
```

1. 从 API 获取的文章都带时间戳，三种模式获取的数据量一致（从 API 取最多 20 篇，足够覆盖时间段需求）
2. `latest` 模式：返回前 1 篇
3. `latest_n` 模式：返回前 N 篇（现有行为）
4. `period` 模式：根据 `period` 计算起始日期，在内存中过滤时间戳符合条件的文章

**文件**: `src/app.py`

- `_resolve_blogger_urls()` 接收并透传 `mode` + 对应参数
- WebSocket 任务入口解析前端传来的 `mode`、`count`、`period`

`refresh_all()` / `_refresh_one_via_api()` 不改动，始终取最新 1 篇。

### 前端改动

**文件**: `src/templates/index.html`

在任务启动区域增加模式选择 UI：

```
[最新一篇]  [最新 N 篇 ▼5]  [时间段 ▼最近一周]
```

- `latest` 模式：无需额外参数
- `latest_n` 模式：显示数字输入框（默认 5，范围 1-20）
- `period` 模式：显示下拉选择（今天/最近3天/最近一周/最近一月）

选择后随 WebSocket 消息发送给后端。

### 不改动的部分

- `POST /api/blogger/refresh` — 始终最新 1 篇
- `POST /api/blogger/{id}/articles` — 保持现有 count 参数
- 手动粘贴 URL 模式 — 不受影响
- `scraper.py` — 不涉及

## 二、问财增强模块

### 性能基准

基于实测数据（2026-04-28）：

| 对比维度 | 现有 StockService | pywencai 问财 |
|---------|-------------------|---------------|
| 单次查询耗时 | 0.05–0.5s | 0.9–2.6s |
| 查询方式 | 指定股票代码 + 单一维度 | 自然语言，全市场筛选 |
| 覆盖范围 | 单只股票 | 一次全市场 5000+ 股票 |
| 数据维度 | 每接口 1 个维度 | 一次 9–20+ 列，融合面面 |
| 适用场景 | 个股深度透视 | 条件选股、板块扫描、事件发现 |

结论：互补关系。现有接口做个股深度，问财做全市场发现。

### 模块一：条件选股

**入口**: 顶部导航新增"条件选股"tab，独立模块

**两种交互**:
- 自然语言搜索框 — 直接输入中文条件，如"市盈率<15 银行股"
- 结构化筛选器 — 行业/概念下拉 + PE/MC/涨跌幅滑块拼装，背后转成问财查询

**结果展示**: 表格列表（代码、简称、最新价、涨跌幅、PE、市值等列），支持排序

### 模块二：板块概念扫描

**入口**: 条件选股模块内或独立 tab

**热力图概览**（俯视全市场）:
- 以概念/行业为卡片或色块
- 颜色深浅表示涨跌幅，大小表示成交额或资金流向
- 点击具体概念 → 进入成分股列表

**概念查询**（深入单个概念）:
- 输入或选择概念词
- 展示该概念所有成分股，按涨跌幅/资金/PE 排序

### 模块三：机构调研

**个股维度**（嵌入个股详情）:
- 查某只股票时，下方展示近期机构调研记录
- 字段：调研日期、机构名称、调研人员、调研地点

**全市场扫描**（条件选股内）:
- 条件选股增加"机构调研"类筛选条件
- 如"近一月有机构调研、股价上涨"
- 结果列表含调研相关信息列

### 模块四：博主观点交叉验证（预留钩子）

本期不做实现，但预留扩展点：
- 文章提取的股票名称/概念关键词 → 自动触发问财查询
- 生成"博主观点 vs 市场数据"对比视图
- `StockService` / `IWencaiService` 和前端组件设计时考虑此扩展

### 后端架构

新增 `src/iwencai_service.py`，与现有 `StockService` 并列：

```
StockService (现有)          IWencaiService (新增)
├── search_stock             ├── query()              — 自然语言选股
├── get_realtime_quote       ├── get_sectors()        — 板块热力图数据
├── get_kline                ├── get_sector_stocks()  — 概念成分股
├── get_financial            ├── get_visits_search()  — 机构调研(全市场扫描)
├── get_money_flow           └── get_stock_visits()   — 机构调研(个股)
├── get_news
└── ...

新增路由:
  POST /api/iwencai/query             — 条件选股
  GET  /api/iwencai/sectors           — 板块热力图
  GET  /api/iwencai/sector/{name}     — 概念成分股
  GET  /api/iwencai/visits/{symbol}   — 个股机构调研
  POST /api/iwencai/visits/search     — 机构调研扫描
```
