# 多市场重构设计

> **⚠️ 部分过时**：2026-05-12 市场模块精简中已删除 `crypto.py` 和 `futures.py`，
> 仅保留 `fund.py`（基金/ETF）。加密货币和期货相关设计不再适用。
> 当前 `market/` 目录仅含 `base.py` + `fund.py`。

## 背景

当前实现（Task 9 完成的版本）存在两个核心问题：

1. **数据加载慢**：基金 ETF 全量数据 `ak.fund_etf_spot_em()` 需要 17.7 秒，缓存 TTL 只有 30 秒
2. **架构不符合预期**：用户期望三个独立侧边栏（股票/基金/币圈），而非当前六个市场 tab 嵌套在板块扫描内

## 变更范围

| 类型 | 内容 |
|------|------|
| **保留不动** | 博主管理、文章收集、配置设置、现有 A 股行情查询 |

| **重构** | "股票查询"增强为多市场统一搜索 |
| **新增** | "基金"侧边栏、"币圈"侧边栏 |

## 二、股票查询增强

### 2.1 搜索逻辑

统一搜索框，自动识别市场：

| 输入特征 | 识别为 | 示例 |
|----------|--------|------|
| 6 位数字，6 开头 | A 股（沪） | 600519 → 贵州茅台 |
| 6 位数字，0/3 开头 | A 股（深） | 000001 → 平安银行 |
| 5 位数字 | 港股 | 00700 → 腾讯控股 |
| 英文字母 | 美股 | AAPL → Apple |
| 中文/混合 | 全市场模糊搜索 | "茅台" → A 股结果 |

### 2.2 数据源

| 市场 | 搜索列表 | 实时行情 | 速度 |
|------|----------|----------|------|
| A 股 | `ak.stock_info_a_code_name()`（已有，5000+只） | 腾讯 API `qt.gtimg.cn` | 搜索 <1s，行情 0.05s |
| 美股 | 精选 300 只（代码+名称静态列表） | 腾讯 API `qt.gtimg.cn` | 搜索 <0.1s，行情 0.05s |
| 港股 | 精选 300 只（代码+名称静态列表） | 腾讯 API `qt.gtimg.cn` | 搜索 <0.1s，行情 0.05s |

### 2.3 股票详情页

点击搜索结果后复用现有股票详情页（行情、K 线、公司简介、财务指标、资金流向、新闻、股东），**完全不变**。

### 2.4 修改文件

**`src/stock_service.py`**：
- 新增 `_US_STOCKS` 静态列表（精选 300 只美股：代码+名称）
- 新增 `_HK_STOCKS` 静态列表（精选 300 只港股：代码+名称）
- 修改 `search_stock()`：识别市场 → 分别查 A 股/美股/港股 → 合并返回
- 新增 `get_us_quote()` 和 `get_hk_quote()`：调用腾讯 API，格式与现有 `_parse_tencent_quote` 一致

**`src/app.py`**：
- 修改 `/api/stock/quote/{symbol}` 路由，支持美股港股行情（当前只支持 A 股）
- 现有 A 股路由不受影响

**`src/templates/index.html`**：
- 股票查询搜索框 placeholder 改为 "搜索 A 股/美股/港股..."
- 搜索结果表格增加"市场"列标识

## 三、基金侧边栏

替代原来的"条件选股"。

### 3.1 功能

- **ETF 板块分类**：按关键词分为 11 类（科技、医药、消费、新能源、金融、军工、资源、宽基、跨境、债券、商品）
- **点击板块**：查看成分 ETF 列表
- **基金搜索**：按代码/名称搜索

### 3.2 数据源

- AKShare `fund_etf_spot_em()` — 1446 只 ETF
- 实时行情列：代码、名称、最新价、涨跌幅、成交额、换手率

### 3.3 性能优化

| 缓存 key | 当前 TTL | 新 TTL | 原因 |
|----------|----------|--------|------|
| `market:fund:boards` | 86400s | 86400s | 不变，板块分类不会变 |
| `market:fund:board:{name}` | 30s | 300s | 减少 17s API 调用频率 |
| `market:fund:search:{kw}` | 30s | 300s | 同上 |
| `market:fund:dataframe` | 无 | 300s | 新增：共享底层 DataFrame，避免重复调用 |

### 3.4 修改文件

**`src/market/fund.py`** — 重写：
- 新增 `_get_etf_df()` 函数：获取并缓存 DataFrame（TTL 300s）
- `get_boards()` 和 `get_board_stocks()` 都调用 `_get_etf_df()` 而不是各自独立调用 AKShare
- 结果缓存 TTL 从 30s 提升到 300s

**`src/app.py`**：
- 原 `/api/iwencai/*` 路由保留（可能有其他用途），但前端不再直接调用
- 新增基金相关路由（或复用现有 market 路由）

**`src/templates/index.html`**：
- 新增"基金"侧边栏按钮 + `view-fund` 视图
- 视图内容：板块网格 + 成分 ETF 表格 + 搜索框

## 五、性能优化

### 5.1 基金数据预加载

在 `app.py` 启动时后台预加载基金 DataFrame：

```python
@app.on_event("startup")
async def startup():
    # 后台预加载基金数据
    asyncio.create_task(_preload_fund_data())
```

预加载函数调用 `ak.fund_etf_spot_em()` 并存入缓存，用户首次访问基金页面即秒开。

### 5.2 缓存策略汇总

| 数据类型 | TTL | 说明 |
|----------|-----|------|
| A 股搜索列表 | 86400s | 静态代码表，日更新 |
| 美股/港股搜索列表 | 86400s | 静态精选列表 |
| 股票实时行情 | 30s | 腾讯 API，快 |
| 基金板块分类 | 86400s | 分类不变 |
| 基金成分 ETF | 300s | 5 分钟更新 |
| 基金搜索 | 300s | 5 分钟更新 |
| 基金底层 DataFrame | 300s | 共享，避免重复请求 |
| 币种行情 | 30s | CoinGecko，保持实时 |

## 七、扩展性

新增一个市场只需：
1. 在 `stock_service.py` 添加市场代码列表
2. 在 `search_stock()` 添加识别逻辑
3. 确认腾讯 API 支持该市场行情（格式前缀如 `us`、`hk`）
4. 前端搜索结果增加市场标签
