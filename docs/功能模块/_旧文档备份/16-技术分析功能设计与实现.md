# 16-技术分析功能设计与实现

## 概述

「技术分析」是 StockPulse 的独立功能模块，提供三项核心能力：

1. **K 线画图工具** — 趋势线、水平线、文字标注、矩形框/通道，与个股+周期绑定持久化
2. **K 线技术形态自动识别** — 十字星、孕线、锤头、射击之星、多头/空头吞噬、MACD 金叉/死叉/顶背离/底背离
3. **分析笔记** — 与个股关联，支持标签，用于复盘

入口：侧边栏独立导航项（`data-view="analysis"`）+ 股票查询 K 线区域「技术分析」快捷按钮。

## 技术形态识别（`services/indicators.py`）

### 蜡烛图形态（`detect_candle_patterns`）

纯 NumPy 实现，无 talib 依赖。

| 形态 | `pattern_type` | 规则 | 颜色 | 位置 |
|------|----------------|------|------|------|
| 十字星 Doji | `doji` | `body / (range + ε) < 0.1` | `#f59e0b` | belowBar |
| 孕线 Harami | `harami` | 当根实体完全在前根实体内（前根为长实体，>1%收盘价） | `#8b5cf6` | aboveBar |
| 多头吞噬 | `bullish_engulf` | 前阴后阳，当根阳线实体完全覆盖前根阴线实体 | `#22c55e` | belowBar |
| 空头吞噬 | `bearish_engulf` | 前阳后阴，当根阴线实体完全覆盖前根阳线实体 | `#ef4444` | aboveBar |
| 锤头 Hammer | `hammer` | 下影线 ≥ 2×实体，上影线 < 实体 | `#22c55e` | belowBar |
| 射击之星 | `shooting_star` | 上影线 ≥ 2×实体，下影线 < 实体 | `#ef4444` | aboveBar |

检测逻辑按优先级顺序（十字星 > 孕线 > 吞噬 > 锤头 > 射击之星），一根 K 线只标注一种形态（`continue` 跳出）。

### MACD 信号（`detect_macd_signals`）

| 信号 | `pattern_type` | 规则 | 颜色 |
|------|----------------|------|------|
| 金叉 | `macd_golden` | `dif[i-1] < dea[i-1]` 且 `dif[i] >= dea[i]` | `#22c55e` |
| 死叉 | `macd_death` | `dif[i-1] > dea[i-1]` 且 `dif[i] <= dea[i]` | `#ef4444` |
| 顶背离 | `macd_bearish_div` | 相邻价格高点：`cls[pj] > cls[pi]` 但 `dif[pj] < dif[pi]` | `#f43f5e` |
| 底背离 | `macd_bullish_div` | 相邻价格低点：`cls[tj] < cls[ti]` 但 `dif[tj] > dif[ti]` | `#06b6d4` |

背离检测依赖 `_find_peaks` / `_find_troughs` 辅助函数（`min_distance=5`，窗口半径为 5 根 K 线），仅在有效 DIF 数据点 ≥ 10 时执行。

## Canvas 叠加层机制

### pointer-events 策略

| 状态 | canvas.style.pointerEvents | 效果 |
|------|---------------------------|------|
| 无工具 / 选择模式 | `none` | 鼠标事件透传给 LightweightCharts，图表正常缩放/十字光标 |
| 激活画图工具 | `auto` | 拦截鼠标事件，图表缩放暂停 |

### 坐标映射

```javascript
// 图表坐标 → 屏幕像素
taTimeToX(timeStr)  // chart.timeScale().timeToCoordinate(timeStr)
taPriceToY(price)   // series.priceToCoordinate(price)

// 屏幕像素 → 图表坐标
taXToTime(x)   // coordinateToLogical(x) → round → _taKlineData[idx].date
taYToPrice(y)  // series.coordinateToPrice(y)
```

画线坐标以价格/时间字符串存储（不存像素），保证缩放/平移后重绘正确。

### 重绘触发时机

1. 鼠标移动时（预览正在绘制的线）
2. 图表缩放/滚动（`subscribeVisibleLogicalRangeChange` 回调）
3. 容器 resize（ResizeObserver → `taResizeCanvas()` + `taRedrawCanvas()`）
4. 画线增删改后

### 命中检测（`taHitTest`）

| 类型 | 判断方式 | 阈值 |
|------|---------|------|
| `hline` | `|mouseY - priceToY(price)| < 8px` | 8px |
| `trendline` | 点到线段欧氏距离（`_taSegDist`） | 8px |
| `text` | 文字包围盒检测 | — |
| `rect` | 矩形四条边逐一检测 | 8px |

双击已选中画线 → 删除；右键 → context menu 提供删除选项。

### 画线持久化流程

```
新建：鼠标抬起 → POST /api/analysis/drawings → 后端返回真实 id → 替换临时 'tmp_xxx' id
删除：若 id 为数字（已持久化）→ DELETE /api/analysis/drawings/{id}
加载：taLoadDrawings() 在 taRenderChart() 之后调用，period 变更时重新加载
```

## 关键设计决策

### 为何选择 Canvas 叠加而非 LightweightCharts ISeriesPrimitive API

LightweightCharts v4 的 `ISeriesPrimitive` 需要理解内部渲染管线，且无法在单 HTML 文件中零构建集成。Canvas 叠加方案：
- 零依赖，坐标映射 API 已在项目中验证
- `pointer-events` 切换策略保持图表正常交互
- 画线以价格/时间存储（非像素），缩放后自动正确

### 为何画线坐标不存像素

LightweightCharts 的时间轴和价格轴会随用户缩放/滚动变化，像素坐标在重绘时将失效。存储价格（float）和时间字符串（`YYYY-MM-DD`），每帧重绘时通过 `priceToCoordinate` / `timeToCoordinate` 实时转换为当前像素，确保画线始终对齐 K 线数据。

### 形态识别不调用 talib 的原因

项目依赖项中无 talib，且 talib 在 Windows 环境安装复杂。纯 NumPy 实现的规则逻辑透明、可维护，满足当前需求精度。
