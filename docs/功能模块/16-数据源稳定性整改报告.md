# 数据源稳定性整改报告

> 日期:2026-06-06
> 范围:http_client、stock_utils(缓存)、stock_service(资金流)、financial_service(财报)、前端 index.html

## 一、问题回顾

用户报告:AKShare 数据经常不及时/取不到、`RemoteDisconnected('Remote end closed connection without response')` 频发,多源降级"不管用"。

## 二、根因诊断(基于真实代码+网络探测)

### 网络层(你这条线 — 杭州电信 36.24.79.139)
| 数据源域名 | 状态 | 备注 |
|---|---|---|
| `web.ifzq.gtimg.cn`(腾讯行情) | ✅ 通 | 报价/K线主力源 |
| `push2.eastmoney.com`(东财单股) | ✅ 通 | `/api/qt/stock/get` 可用 |
| `push2.eastmoney.com`(东财列表) | ❌ TCP RST | clist/get 接口不可用 |
| `82.push2.eastmoney.com` | ❌ TCP RST | 整段不可用 |
| `push2his.eastmoney.com` | ❌ TCP RST | 资金流日线接口在这里 |
| `datacenter-web.eastmoney.com` | ✅ 通 | 财报数据接口 |
| `d.10jqka.com.cn`(同花顺) | ✅ 通 | K线备选 |
| `hq.sinajs.cn`(新浪) | ✅ 通(需 Referer) | 备选 |

**重要结论**:东财部分子域名在你当前网络层就被 TCP RST,代码层无法救——除非换网络/代理。

### 代码层(三个结构性问题)
1. **全局 patch 锁**:`http_client.patch_requests` 用全局锁包整个 AKShare 调用 → 所有 AKShare 请求强制串行
2. **共享 Session 无 Retry + 无浏览器 UA** → 死连接复用 → RemoteDisconnected;反爬接口直接被拒
3. **降级覆盖不全**:race_sources 只用于报价/K线,资金流/财报/股东等是裸调用,一失即败
4. 缓存过期就报错,无 stale-while-revalidate 兜底

## 三、整改清单(已落地)

### 1. `http_client.py` 重写
- ✅ `requests.Session` 挂 **Retry 适配器**(`total=2, backoff=0.4, status=429/500/502/503/504`)
- ✅ 默认 **浏览器 UA + Accept 头**(过反爬门槛)
- ✅ `patch_requests` 改用 **线程局部 patch**(去掉全局锁,AKShare 调用可真正并发)
- ✅ `httpx.AsyncClient` 也带浏览器 UA + 显式 limits + connect 超时
- ✅ 保留代理分流:国内通路 `trust_env=False`,国外 AI 通路 `trust_env=True`

### 2. `stock_utils.py` 新增 stale-while-revalidate
- ✅ `cache_get_stale(key, max_stale_seconds=7d)` → 返回 `(value, is_fresh)`
- ✅ 不破坏现有 `cache_get`/`cache_set`

### 3. `stock_service.get_money_flow` 多源 + 陈旧兜底
- ✅ race:东财 push2his 直连 + AKShare 兜底(整体超时 6s)
- ✅ 失败时:`cache_get_stale` 7 天内的旧值,带 `stale=True` 标记
- ✅ 全失败时:返回明确文案 *"东方财富在当前网络环境暂不可达,可尝试切换网络/代理后重试"*

### 4. `financial_service.fetch_indicators`(刚加的财报模块)
- ✅ 取数加重试 2 次 + 退避
- ✅ 失败时同样走 stale 兜底

### 5. 前端 `index.html`
- ✅ 资金流卡片加 stale 黄色横幅 *"⚠ 显示的是缓存数据,实时数据源暂不可达"*
- ✅ 错误状态改成紧凑小字,不再撑卡片

## 五、已知限制(给你回来知情)

| 项 | 状态 | 说明 |
|---|---|---|
| A 股资金流 | **网络层不通** | 你这条线访问 `push2his.eastmoney.com` 被 TCP RST。代码层已加 race + stale 兜底;只要曾经成功过一次,后续即使数据源挂了也能显示旧值。彻底解决需要切网络/换代理路由。 |
| `push2.eastmoney.com` clist | 不通 | 影响:大盘涨跌家数等(cockpit_service 里已有 3 次重试,但仍可能慢) |
| `push2his`(K 线/资金流) | 不通 | K 线已经有腾讯/mootdx 备选,不受影响;资金流没有等价备选 |

**建议**:你 VPN 规则里加东财域名 *直连* 规则(`*.eastmoney.com` → DIRECT),或者切换到不被 RST 的网络环境。代码这边已经做到最大化的鲁棒性。
