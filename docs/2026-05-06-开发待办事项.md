## Phase 1：产品可用化（3 周）

> **目标**：让产品从"演示 Demo"变成"日常能用的工具"。数据持久化是后续所有功能的地基。

### 1.1 数据持久化（先行依赖，Week 1）

- [ ] **P0** 📊 M · SQLite 数据库模块 `src/database.py`
  - 统一连接管理（单例），WAL 模式
  - 建表：`bloggers` / `articles` / `cache`（通用 KV 缓存，替代内存 TTLCache）
  - 不引入 ORM——直接用 `sqlite3` 标准库，保持零依赖
  - 验收：现有 JSON 数据可迁入、重启后数据不丢失

- [ ] **P0** 📊 S · 数据迁移脚本
  - `bloggers.json` → `bloggers` 表
  - 内存缓存 → `cache` 表（行情、财务数据持久化）
  - 验收：迁移后功能无变化

### 1.2 Watchlist 模块（Week 2）

- [ ] **P1** ⚡ M · Watchlist 后端 API
  - 表：`watchlist`（symbol / market / added_at / note / sort_order）
  - 接口：GET（列表 + 实时行情）、POST（添加）、DELETE（删除）、PUT（排序、备注）
  - 验收：可管理 ≥ 50 只标的，行情实时刷新

- [ ] **P1** 🎨 S · Watchlist 前端
  - 侧栏新 Tab：自选
  - 表格式展示：代码 / 名称 / 现价 / 涨跌幅 / 市值
  - 涨跌排序、拖拽排序
  - 验收：加删改查 + 实时刷新

### 1.3 技术指标（Week 3）

- [ ] **P1** ⚡ M · 基础技术指标实现
  - RSI、MACD、KDJ、BOLL（纯 NumPy 实现，不依赖 talib）
  - 验收：与同花顺/东财结果对得上

- [ ] **P1** 🎨 S · K 线图叠加指标
  - 前端勾选面板：开关 RSI / MACD / KDJ / BOLL
  - MACD 副图、RSI 副图、KDJ 副图、BOLL 主图叠加
  - 验收：指标线显示正确、开关流畅

### 1.4 工程穿插（不单独排期，随功能修改时顺手做）

- [ ] **P2** 🔧 M · 拆分 `app.py`（渐进式，不一次拆完）
  - 每次改某个路由时，把那个路由模块拆到 `src/routers/`
  - 最终 app.py < 300 行

- [ ] **P2** 🔧 S · 核心模块测试（只测数据正确性）
  - `stock_service.py` 行情解析（5-8 个 case）
  - `stock_utils.py` 缓存与代码识别（3-5 个 case）
  - 不做覆盖率目标，只保证核心逻辑可信

- ❌ 求职物料（白皮书、Demo 视频、博客、求职话术）——产品导向不需要

- ❌ QuantLib / Qlib / RDAgent 量化框架
- ❌ HFT / 强化学习交易
- ❌ 16+ broker 实盘集成
- ❌ 节点编辑器 / 代码编辑器内嵌
