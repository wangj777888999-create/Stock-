## 1. 项目定位

### 1.1 一句话描述

**StockPulse — 个人投资信息平台**：监控微信公众号博主 → 抓取文章 → AI 多视角分析 → 跟踪推荐股票 → 模拟/实盘交易 → 胜率统计

差异化：三视角 AI 分析（价值派/成长派/趋势派）+ 交易感知闭环 + MCP 协议

### 1.2 目标用户

| 用户群 | 痛点 | 方案 |
|---|---|---|
| 中文资深散户 | 微信公众号信息散乱、缺少结构化 | 博主聚合 + 自动抓取 + AI 分析 |
| 多市场配置者 | A/H/U/基金分散在 N 个 App | 一界面切换全市场 |
| AI 重度使用者 | 通用 AI 无金融语境 | MCP 工具 + 三视角 Prompt |
| 求职 PM / 开发者 | 需要可展示的金融产品作品集 | 这就是作品集本身 |

### 1.3 解决的痛点

| 痛点 | 方案 |
|------|------|
| 关注的股票博主分散在不同公众号，手动翻看效率低 | 批量监控博主，一键抓取最新文章 |
| 文章太多没时间逐篇读 | AI 自动分析文章要点，支持多投资视角 |
| 博主推荐的股票涨跌无法追踪验证 | 博主喊单记录 + 事后验证，用数据说话 |
| 想回测某个想法但没有工具 | 内置模拟交易系统，10万虚拟本金 |
| 行情数据分散在不同 App | 一个平台覆盖 A/H/U/KR/JP + 基金 |
| 各种财经 App 广告多、数据不透明 | 本地部署，零广告，数据来源完全可控 |

## 3. 技术架构

### 3.1 架构图

```
前端 SPA (templates/index.html，~5150 行)
  → FastAPI app.py (176 行，纯组装)
    → routers/ (12 个路由模块)
      → 业务服务层 (stock_service / sector_service / blogger / analyzer 等)
        → SQLite database.py (WAL 模式，单连接，11 张表)
        → L1 内存缓存 + L2 SQLite 缓存 (stock_utils.py)
        → 外部 API (AKShare / 腾讯 / yfinance / 东方财富 / pywencai)
      → services/ (eastmoney.py / source_racer.py / indicators.py)
```

### 3.2 后端技术栈

| 类别 | 技术 |
|---|---|
| Web 框架 | FastAPI / Uvicorn / Pydantic |
| 异步 | asyncio + Semaphore + asyncio.to_thread |
| 爬虫 | Playwright 1.40+ / BeautifulSoup4 / requests |
| 金融数据 | AKShare / yfinance / pywencai / pandas |
| 数据库 | SQLite（WAL 模式，单连接，11 表） |
| AI | httpx + OpenAI 兼容 API |
| 邮件 | smtplib / email.mime / markdown |
| HTTP 客户端 | requests.Session（AKShare）+ httpx.AsyncClient（异步直连） |

### 3.3 前端技术栈

| 维度 | 实现 |
|---|---|
| 构建 | 无打包，纯 HTML + 内联 CSS + 内联 JS |
| 文件 | 单文件 `index.html`（~5150 行） |
| 图表 | LightweightCharts.js |
| 设计系统 | CSS 变量 tokens |
| 实时通信 | 原生 WebSocket + JSON |

### 3.4 部署

- 启动：`python app.py`（在 `wexin-read-mcp-main/src/` 下）
- MCP：独立进程 `python server.py`
- 存储：SQLite `data.db`（主存储）+ `bloggers.json`（博主列表）+ `user_config.json`（配置）

### 3.5 安全设计

- 邮箱密码 / API Key / 微信 Cookie **永不持久化**，仅环境变量读取
- API 响应自动脱敏
- WebSocket URL 白名单

## 4. 完整能力清单

### 4.1 数据采集

#### 微信文章采集
- **文件**：`src/scraper.py`（234 行）+ `src/parser.py`（74 行）
- **方案**：Playwright 浏览器自动化 + 反检测（隐藏 webdriver 标志、自定义 UA）
- **可靠性**：3 次重试 + SIGPIPE / 网络超时检测
- **统一存储**：`article_storage.py`（106 行）

#### 博主信息识别
- **文件**：`src/blogger.py`（723 行）
- **方案**：从公众号文章 URL 反向识别博主（biz / user_name / avatar / name）
- **双源降级**：mp_backend API → getmsg API 自动故障转移

#### A 股市场数据
- **文件**：`src/stock_service.py`（1011 行）
- **数据矩阵**：

| 数据类型 | 数据源 | 字段说明 |
|---|---|---|
| 实时行情 | 腾讯股票 API | 16 项：实时价/涨跌幅/成交量/市值/换手率/振幅等 |
| K 线 | AKShare（主）+ 腾讯（备） | 日/周/月线前复权，730 天历史 |
| 公司信息 | AKShare 巨潮 + 东方财富降级 | 双源容灾 |
| 财务数据 | AKShare 同花顺 + 东方财富降级 | 10+ 指标 |
| 资金流向 | AKShare 东方财富 | `stock_individual_fund_flow` |
| 个股新闻 | AKShare | `stock_news_em` |
| 公告 | 巨潮资讯接口 | `stock_notice_report` |

#### A 股板块行情
- **文件**：`src/sector_service.py`（590 行）
- **数据源**：东方财富（主）→ 同花顺（备）→ pywencai（第三备）
- **能力**：行业/概念板块列表、成分股（含 PE/市值/换手率）、板块 K 线（日/周/月，2 年历史）
- **热力图**：按涨跌幅排序，7 色阶 CSS Grid 均匀网格

#### 全球市场

| 市场 | 文件 | 数据源 | 精选数量 |
|---|---|---|---|
| 港股 | stock_service.py | 腾讯 API | 30 只龙头 |
| 美股 | stock_service.py | 腾讯 API | 100+ 只 |
| 韩股 | global_stock_service.py | yfinance | 20 只 |
| 日股 | global_stock_service.py | yfinance | 20 只 |

#### 基金 / ETF
- **文件**：`src/market/fund.py`
- **数据源**：AKShare `fund_etf_spot_em`
- **分类**：11 类（科技/医药/消费/新能源/金融/军工/资源/宽基/跨境/债券/商品）

#### 同花顺问财
- **文件**：`src/iwencai_service.py`（200 行）
- **能力**：自然语言条件选股、板块热力图、机构调研、板块成分股

#### 东方财富直连（降级源）
- **文件**：`src/services/eastmoney.py`（142 行）
- **能力**：公司基本信息 + 最近 8 期核心财务指标

#### 多源竞速工具
- **文件**：`src/services/source_racer.py`（106 行）
- **能力**：并发启动所有数据源，取最先成功返回的结果；EWMA 延迟追踪 + 失败惩罚排序

### 4.2 数据处理与分析

#### K 线处理
- 日 → 周 / 月聚合、MA 自定义周期 1-240、前复权处理
- 技术指标：RSI / MACD / KDJ / Bollinger（`services/indicators.py`）

#### 财务指标高亮（规则引擎）
- 配置：`src/financial_rules.json`
- 规则：净利润同比 / 资产负债率 / ROE / 毛利率 / EPS / 净利润
- 颜色：red（异常）/ green（优秀）/ orange（警示）

#### AI 投资分析
- **文件**：`src/analyzer.py`（317 行）+ `src/agents/personas.py`
- **三大投资视角**：

| 视角 | 关注点 | Prompt 关键词 |
|---|---|---|
| **价值派**（buffett） | 护城河、ROE、自由现金流、估值 | PE / PB / 股息率 |
| **成长派**（growth_lynch） | 营收/利润增速、PEG、行业渗透率 | 产品迭代、市场空间 |
| **趋势派**（trend_follower） | 板块轮动、量价配合、强势股梯队 | 技术信号 |

- **并行分析**：多视角同时运行，**AI 接入**：兼容任何 OpenAI 格式 API

### 4.3 交易感知闭环

| 阶段 | 功能 | 路由 |
|---|---|---|
| 推荐追踪 | 博主喊单记录 | `routers/verify.py` |
| 模拟交易 | 模拟买卖 + 持仓管理 + 回测 | `routers/sim.py` |
| 实盘记录 | 真实交易流水记录 | `routers/verify.py` |
| 交易日记 | 投资心得、操作复盘 | `routers/journal.py` |
| 胜率统计 | 博主推荐准确率、盈亏比分析 | `routers/stats.py` |

### 4.4 服务形态

#### Web 应用（FastAPI）
- 前端：纯 HTML 单文件（~5150 行），9 个页面
- 路由：12 个独立路由模块
- 图表：LightweightCharts.js
- 实时：WebSocket + JSON 协议

#### MCP 服务器
- **文件**：`src/server.py`（92 行）
- **框架**：fastmcp
- **暴露工具**：`read_weixin_article(url: str)`

#### 定时调度
- **文件**：`src/scheduler.py`（184 行）
- **能力**：定时采集博主文章，支持 latest_n / since_date / period 三种模式

### 4.5 缓存与性能

| 维度 | 实现 |
|---|---|
| L1 缓存 | 进程内 dict（`_mem`），命中无磁盘 I/O，< 0.1ms |
| L2 缓存 | SQLite `cache` 表，持久化，重启不丢失，5~20ms |
| TTL 等级 | REALTIME_REFRESH 5s / REALTIME 30s / DAILY 300s / KLINE 3600s / BOARDS 1800s / COMPANY 86400s |
| 启动预热 | 基金/板块/自选股并发预拉取 |
| HTTP 连接池 | requests.Session + httpx.AsyncClient，`trust_env=False` 绕 VPN |
| 并发限流 | SemaphorePool（最多 3 并发博主获取） |

## 7. 快速开始

### 环境要求

- Python 3.10+
- macOS / Linux / Windows

### 安装与启动

```bash
# 1. 克隆项目
git clone <repo-url>
cd wexin-read-mcp-main

# 2. 安装依赖
pip install -r requirements.txt

# 3. 安装 Playwright 浏览器（文章抓取需要）
playwright install chromium

# 4. 启动服务
cd src
python app.py

# 5. 打开浏览器
# http://localhost:8000
```

### 配置

启动后进入「配置设置」页面：

| 配置项 | 说明 |
|--------|------|
| SMTP 邮件 | 填写发件邮箱，用于接收 AI 分析报告 |
| AI API | 填写 API Key、Base URL、模型名，用于文章分析 |
| 微信 Cookie | 扫码登录或手动粘贴，用于抓取公众号文章 |

也可通过环境变量配置（优先级高于配置文件）：

```bash
export SMTP_SERVER=smtp.163.com
export SMTP_EMAIL=your@163.com
export SMTP_PASSWORD=your_password
export AI_API_KEY=sk-xxx
export AI_BASE_URL=https://api.openai.com/v1
export AI_MODEL=gpt-4o-mini
```

### 作为 MCP 工具使用（Claude Desktop）

```json
{
  "mcpServers": {
    "weixin-reader": {
      "command": "python",
      "args": ["/path/to/src/server.py"]
    }
  }
}
```
