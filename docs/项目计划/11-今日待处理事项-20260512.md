## 任务一：前端内存泄漏修复（阶段二）

**文件**：`wexin-read-mcp-main/src/templates/index.html`
**预估工时**：2-3 小时
**背景**：前端重构阶段一（安全修复）已完成，阶段二是内存泄漏修复。长时间使用会导致页面变慢，是影响日常使用体验的实际问题。

### 6 处泄漏清单

| # | 类型 | 行号 | 问题描述 | 修复方式 |
|---|------|------|---------|---------|
| L1 | ResizeObserver | **L3526** | MACD 图表 observer 直接 `new` 无变量保存，无法 disconnect | 提取为 `_klineMacdObserver` 变量，切换时 disconnect |
| L2 | ResizeObserver | **L3561** | RSI 图表 observer 同上 | 提取为 `_klineRsiObserver` 变量，切换时 disconnect |
| L3 | ResizeObserver | **L4303** | `switchFundKlinePeriod()` 内每次切换周期都创建新 observer，旧的不释放 | 提取为 `_fundResizeObserver` 变量，重新创建前先 disconnect |
| L4 | ResizeObserver | **L4357** | `selectFund()` 内同上 | 同上，复用 `_fundResizeObserver` 变量 |
| L5 | ResizeObserver | **L4554** | `switchFuturesKlinePeriod()` 内同上 | 提取为 `_futuresResizeObserver`，重新创建前先 disconnect |
| L6 | ResizeObserver | **L4612** | `selectFutures()` 内同上 | 同上，复用 `_futuresResizeObserver` 变量 |

> **注**：`_watchlistInterval`（L5123）已有 `if (_watchlistInterval) return` 防重复保护，但切换视图时不清理——如有余力可顺手加，参见下方附加项。

### 修复方案

**第一步**：在全局变量区（L3270 附近）补充声明：

```javascript
let _klineMacdObserver = null;
let _klineRsiObserver = null;
let _fundResizeObserver = null;
let _futuresResizeObserver = null;
```

**第二步**：把 L3526、L3561 的匿名 `new ResizeObserver(...)` 改为保存到对应变量，并在 K 线加载函数顶部（`loadKline()` 内，L3425 附近已有 `_klineResizeObserver.disconnect()` 逻辑）插入同样的 disconnect：

```javascript
// 在重建 MACD/RSI observer 前先清理
if (_klineMacdObserver) { _klineMacdObserver.disconnect(); _klineMacdObserver = null; }
if (_klineRsiObserver) { _klineRsiObserver.disconnect(); _klineRsiObserver = null; }

// 然后保存新 observer
_klineMacdObserver = new ResizeObserver(() => { ... });
_klineMacdObserver.observe(macdEl);

_klineRsiObserver = new ResizeObserver(() => { ... });
_klineRsiObserver.observe(rsiEl);
```

**第三步**：L4303 和 L4357（基金图表）、L4554 和 L4612（期货图表）同样处理——函数顶部先 disconnect 旧的，再保存新的。

**第四步**：在 `switchView()`（L2307）函数顶部加通用清理：

```javascript
function switchView(view, el) {
  // 离开旧视图前清理副作用
  if (_klineMacdObserver) { _klineMacdObserver.disconnect(); _klineMacdObserver = null; }
  if (_klineRsiObserver) { _klineRsiObserver.disconnect(); _klineRsiObserver = null; }
  if (_fundResizeObserver) { _fundResizeObserver.disconnect(); _fundResizeObserver = null; }
  if (_futuresResizeObserver) { _futuresResizeObserver.disconnect(); _futuresResizeObserver = null; }
  // ... 原有逻辑继续
```

### 验收标准

打开 DevTools → Memory 面板 → 反复切换视图 10 次（股票→基金→期货→股票...），Detached 节点数不持续增长。

### 附加项（可选，顺手做）

`_watchlistInterval` 清理：在 `switchView()` 清理块中加：

```javascript
if (_watchlistInterval && view !== 'watchlist') {
  clearInterval(_watchlistInterval);
  _watchlistInterval = null;
}
```

## 任务二：北向资金 + 龙虎榜

**文件**：后端新建路由或追加到现有路由；前端 `index.html` 新增视图入口
**预估工时**：1-2 小时
**背景**：AKShare 已有现成接口，A 股用户的核心需求，开发成本极低。

### 后端实现

在 `wexin-read-mcp-main/src/routers/market.py` 追加 3 条路由（或单独建 `routers/northbound.py`）：

| 路由 | AKShare 函数 | 说明 |
|------|-------------|------|
| `GET /api/market/north-flow` | `ak.stock_em_hsgt_north_net_flow_in()` | 北向资金实时净流入 |
| `GET /api/market/north-history` | `ak.stock_em_hsgt_hist_fund_flow()` | 北向资金历史趋势 |
| `GET /api/market/dragon-tiger` | `ak.stock_lhb_detail_daily_sina()` | 龙虎榜当日数据（参数：date=今日） |

每条路由套现有的 TTL 缓存（`stock_utils.py` 的 `get_cached`），北向资金用 `300s`，龙虎榜用 `600s`（收盘后数据不变，可给更长值）。

示例路由结构（参照现有 `market.py` 风格）：

```python
@router.get("/api/market/north-flow")
async def get_north_flow():
    try:
        df = ak.stock_em_hsgt_north_net_flow_in(symbol="北向资金")
        return {"success": True, "data": df.tail(1).to_dict(orient="records")}
    except Exception as e:
        return {"success": False, "error": str(e)}
```

### 前端实现

在侧边栏「期货」入口下方新增「北向/龙虎」导航项，对应视图 `view-northbound`，包含两个 tab：
- **北向资金**：今日净流入数字 + 近 30 日折线图（用 lightweight-charts，复用现有图表初始化模式）
- **龙虎榜**：表格展示（代码、名称、买入金额、卖出金额、净买入、上榜原因）

新增代码按 V2 规范（[10-StockPulse-V2设计文档](10-StockPulse-V2设计文档.md)）：
- 数据获取封装为 `window.StockPulse.api.getNorthFlow()` 等独立函数
- 页面逻辑挂在 `window.StockPulse.pages.northbound` 下
- 使用 `loadNorthboundPage()` / `renderNorthboundPage()` load/render 分离

### 验收标准

- 点击「北向/龙虎」导航，北向资金今日净流入数字正常显示
- 龙虎榜表格有数据，非交易日显示"今日暂无龙虎榜数据"而不是报错
