## 1. 核心结论

**差异化定位已清晰：AI-first 中国散户投研工作台。不需要对标 Bloomberg，专注「微信博主 + 多视角 AI + 交易验证闭环」三个核心壁垒。**

### 对后续开发的指导

1. **不要铺摊子**：当前功能完整度已经够高，深度优于广度
2. **近期可追加**：北向资金、龙虎榜（AKShare 有现成接口）
3. **V2 方向**：博主评分排行榜（M2-M5）> AI 分析升级 > 通知推送
4. **不追的**：期权、地缘政治、HFT、QuantLib

## 2. 竞品分析

### 2.1 DSA（daily_stock_analysis）

GitHub 34k+ Stars，LLM 驱动的自动化股票分析系统。

**核心功能：**

| 功能 | DSA | 本项目 |
|------|-----|--------|
| AI 决策看板 | 一行结论 + 0-100 评分 + 操作清单 | 三视角 AI 分析（价值/成长/趋势） |
| 内置策略 | 11 种（多头趋势、缠论、波浪理论等） | 财务规则引擎（JSON 配置） |
| 通知渠道 | 12+（企微/飞书/Telegram/Discord...） | 仅邮件 SMTP |
| 数据源 | 8 个数据源级联回退 | AKShare + 腾讯 + yfinance + pywencai |
| 回测 | 有（历史准确率验证） | 有模拟交易，无策略回测 |
| 前端 | React 19 + TypeScript | 原生 HTML 单文件 |

**DSA 的技术亮点（可借鉴）：**
- YAML 策略 DSL：声明式定义交易策略，无需改代码
- `json-repair` 库：修复 LLM 返回的格式错误 JSON
- `exchange-calendars` 库：自动跳过节假日
- LiteLLM Router：统一接入 15+ LLM 供应商
- 多 Agent 编排：4 种模式（quick/standard/full/specialist）

### 2.2 FinceptTerminal

C++/Qt 桌面应用，52 个功能屏 + 37 个 AI Agent，复刻 Bloomberg Terminal。

**核心差异：**

| 维度 | FinceptTerminal | 本项目 |
|------|----------------|--------|
| 形态 | 桌面应用（1GB+） | 轻量 Web |
| 目标用户 | 量化研究员 / 机构 | 个人投资者 / 中文散户 |
| 本地化 | 浅（中国数据浅层接入） | 深（微信博主 / 问财 / 龙虎榜 / 北向资金） |
| 许可 | AGPL + $10,200/年商业 | 个人工具，无商业压力 |
| 微信博主 | 无 | 独有 |
| 同花顺问财 | 无 | 独有 |

**FinceptTerminal 的差距（不必追）：**
- 外汇/债券、地缘政治 Agent、SEC EDGAR、QuantLib 套件
- 这些在散户场景中需求极弱

### 2.3 竞品对比总结

本项目的差异化不在于数据广度或量化深度，而在于：
1. **微信信息源**（中文投资博主聚合，竞品都没有）
2. **AI 多视角分析**（价值/成长/趋势并行，避免单一偏见）
3. **交易验证闭环**（从推荐到胜率统计的完整链路）

## 4. 技术发现与坑

已沉淀为代码的关键技术问题和解法：

| 问题 | 解法 |
|------|------|
| 微信 MP API 两套接口差异 | mp_backend → getmsg 双源降级 |
| AKShare 被系统代理拦截 | `patch_requests()` + 同时 patch Session 类 |
| 东方财富成分股 ProxyError | 三层降级：东方财富 → 同花顺 → pywencai |
| 股票搜索慢（8.4s） | SQLite 缓存命中后降至 0.005s（1690x） |
| Playwright 反检测 | 隐藏 webdriver + 自定义 UA |
| pywencai 列名含日期后缀 | startswith 匹配重命名 |
| macOS 下 `proxies=None` 延迟 20s | `trust_env=False` 绕过 VPN |
| K 线 count=99999 哨兵值冲突 | 新增 `all_history=true` 显式参数 |
| AKShare 列名版本漂移 | `KLINE_COL_ALIASES` 别名映射 |
| ResizeObserver 内存泄漏 | 保存引用 + disconnect + switchView 兜底 |

## 5. V2 产品规划

### 5.1 技术栈决策

| 层 | 当前 | 未来（预留） |
|---|---|---|
| 前端 | 原生 JS（load/render 分离） | Vite + React 18 + TypeScript |
| UI | CSS 变量 tokens | shadcn/ui + Tailwind CSS |
| 状态管理 | `window.StockPulse.pages.*` | Zustand |
| 后端 | FastAPI + SQLite | 不变 |
| 推送 | 邮件 SMTP | 企微 Webhook + Telegram Bot |

### 5.2 前端迁移就绪原则

当前不迁移 React，但新增代码遵循：
1. **load/render 分离**：`loadXxxPage()`（数据获取）+ `renderXxxPage()`（DOM 渲染）
2. **API 调用独立**：所有后端请求封装在 `window.StockPulse.api` 下
3. **状态集中管理**：页面状态挂在 `window.StockPulse.pages.*`
4. **事件处理分离**：事件绑定在 `load` 阶段统一注册，不内联 onclick
5. **无全局变量泄漏**：新增变量挂在 `window.StockPulse` 命名空间

### 5.3 博主荐股 AI 评分系统（核心差异化）

```
每日自动采集 → 遍历博主 → 抓取文章 → AI 扫描提及股票
→ 文章管理页面（人工确认）→ 创建荐股记录
→ 每日收盘拉取价格 → 计算四项指标 → 综合评分 → 前端排行榜
```

**综合评分算法：**

```
可信度分 = 0.35 × 命中率分 + 0.30 × 风险收益比分 + 0.20 × 跟踪深度分 + 0.15 × 样本量加权

命中率分   = 推荐后 10 日内最大涨幅 > 5% 的占比，归一化到 0-100
风险收益比 = 平均(最大涨幅 / |最大回撤|)，> 3 得满分
跟踪深度分 = 是否有后续文章更新观点（有=100，无=30）
样本量加权 = min(推荐数/20, 1.0)
```

### 5.4 AI 分析升级

- **YAML 策略 DSL**：声明式定义投资策略，无需改代码
- **LLM 容错**：`json-repair` 库修复格式错误的 JSON 响应
- **交易日历**：`exchange-calendars` 库，非交易日跳过行情请求
- **Persona 扩展**：从 3 个扩到 5-7 个（加 Munger / Soros / Klarman 等）

### 5.5 通知与事件监控

**推送渠道（插件架构）：**

```
src/notifications/
├── base.py          # 抽象基类 Notifier
├── email.py         # 现有邮件
├── wecom.py         # 企微机器人（Webhook）
├── telegram.py      # Telegram Bot API
└── dispatcher.py    # 统一调度
```

**事件类型：**

| 事件 | 触发条件 |
|------|---------|
| 价格预警 | 自选股价格穿越预警价 |
| 荐股更新 | 关注博主发布新荐股文章 |
| 涨跌幅异动 | 自选股单日涨跌超 N% |
| 博主评分变化 | 评分升降超阈值 |

### 5.6 风险与缓解

| 风险 | 缓解 |
|------|------|
| 原生 JS 可维护性下降 | 严格 load/render 分离 + 命名空间隔离 |
| AKShare/腾讯 API 变动 | 双源回退 + 数据源错误日志 |
| AI 提取荐股准确率不足 | "AI 提取 + 人工确认" 机制兜底 |
| LLM API 不稳定 | json-repair + 非阻塞降级 |
