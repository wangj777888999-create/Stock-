## 三、技术设计细节

### 3.1 P0-1：mootdx 行情提速 ✅ 已完成

**目标**：A 股 K 线延迟从当前 ~5-15s（AKShare HTTP 串行降级）降至 ~50ms（TCP 直连）。

**实现**：`services/mootdx_provider.py` — 独立 async 模块，使用 `mootdx.quotes.Quotes` TCP 直连。

```python
# services/mootdx_provider.py（已实现）
from mootdx.quotes import Quotes
from mootdx import HQ_HOSTS

def _fetch_sync(symbol, period, count):
    name, ip, port = HQ_HOSTS[0]  # 深圳双线主站
    client = Quotes.factory(market="std", server=(ip, port))
    df = client.bars(symbol=symbol, frequency=freq, start=0, offset=count)
    # 返回 [{date, open, close, high, low, volume}]
```

**实测延迟**：TCP 连接 ~50ms，async 包装后 ~680ms（含 event loop 开销）。

**注意事项**：
- mootdx 不返回 PE/PB/市值/换手率 — 这些仍需腾讯 HTTP 补充
- TCP 7709 端口在部分网络环境下可能被封，超时 8s 后 race_sources 自动降级
- 使用 `mootdx.quotes.Quotes`（非 `mootdx.api.MdApi`，后者在 v0.11 中不存在）
- 本地文件读取模式需要 TDX 数据目录，未使用

**竞速策略**（已实现）：
`AKShare HTTP` ∥ `腾讯 HTTP` ∥ `mootdx TCP` — 三源并发，`race_sources` EWMA 自动排序。

### 3.5 P1-5：消除 http_client 全局锁

当前问题：`http_client.py` 的 `patch_requests` 将全局 `requests.Session` 替换，导致 AKShare 的所有 HTTP 调用被隐式序列化。

解决方案（参考 a-stock-data 用 urllib 绕开的思路）：
1. 将代理绕过逻辑从 Session 级别下沉到 headers/transport 级别
2. 不再 monkey-patch 全局 requests，改为显式传递自定义 transport
3. AKShare 调用走独立 `requests.Session`（无代理 patch）
