## 1. 数据采集与存储

### 1.1 统一存储层（`article_storage.py`）

文章采集有两条路径（WebSocket 手动 + Scheduler 定时），统一由 `article_storage.py` 管理持久化。

**核心函数：**

| 函数 | 说明 |
|------|------|
| `save_articles(articles, blogger_id, ai_mentions, skip_existing)` | 唯一写入入口，`INSERT OR IGNORE` 按 URL 去重 |
| `get_existing_urls(blogger_id)` | 获取已存储 URL 集合，用于去重 |
| `update_mentions(url, mentions)` | 更新 AI 提及数据（Scheduler 路径用） |
| `make_preview(content, length=150)` | 过滤 HTML 标签后截取前 150 字 |

**两条路径差异：**

| 路径 | AI 扫描时序 | blogger_id |
|------|------------|------------|
| WebSocket | 先扫描后存储（`ai_mentions` 随文章写入） | `"__manual__"` |
| Scheduler | 先存储后扫描（`update_mentions` 更新） | 博主实际 ID |

**关键设计决策：**
- 手动采集统一使用 `MANUAL_BLOGGER_ID = "__manual__"` 而非空串
- WebSocket 等待用户勾选有 300 秒超时，超时默认全部存储
- `selected_urls` 缺失（旧版前端）→ 全存；空列表 → 跳过；有值 → 按选中过滤

### 1.2 调度器（`scheduler.py`）

- 支持三种采集模式：`latest_n` / `since_date` / `period`
- 调用 `save_articles()` + `update_mentions()` 统一存储
- 由 `app.py` 的 `_startup()` 启动

## 2. K 线数据防护

> 来源：`06-K线数据断路根因分析与防护方案`

### 2.1 五大故障模式（全部已修复）

| 编号 | 故障 | 根因 | 修复方案 |
|------|------|------|---------|
| F1 | `count=99999` 被 `le=1000` 拒绝 | 哨兵值在 3 处使用，含义不统一 | 新增 `all_history=true` 显式参数 |
| F2 | AKShare 列名硬编码 | 列名随版本漂移 | `KLINE_COL_ALIASES` 别名映射 + `_resolve_col()` |
| F3 | 全 None 记录仍返回 success | `.get()` 静默返回 None + 坏数据入缓存 | 写缓存前验证 OHLC 不全为 None |
| F4 | ResizeObserver 泄漏 | `new ResizeObserver()` 无引用保存 | 保存引用 + `_disposeMainChart()` |
| F5 | Lock 阻塞并发请求 | AKShare 请求 5-10s 持锁 | `asyncio.wait_for(timeout=15)` 超时保护 |

### 2.2 数据链路

```
前端 loadKline(period)
  → GET /api/stock/kline/{symbol}?period=&count=&indicators=
    → routers/stock.py: all_history 参数处理
      → stock_service.py: 缓存检查 → AKShare/腾讯降级
        → 列名别名映射 → OHLC 验证 → 写缓存
        → calc_rsi / calc_macd
      → 返回 {success, data, indicators}
    → 前端: LightweightCharts 渲染 + MACD/RSI 子图
```

### 2.3 迭代安全检查清单

修改 `routers/stock.py` 时：
- [ ] `count` 上限与前端发送值兼容
- [ ] 新增参数后检查前端所有 `fetch` URL

修改 `stock_service.py` 时：
- [ ] AKShare 列名用别名映射而非硬编码
- [ ] 组装 records 后验证 OHLC 不全为 None

修改 `index.html` 时：
- [ ] `loadKline` 调用 `_disposeMainChart()`
- [ ] 新增 ResizeObserver 保存引用并在下次 disconnect

升级 AKShare 时：
- [ ] 运行列名检查脚本确认未变

## 3. API 性能优化（双层缓存）

> 来源：`14-API性能优化实施计划`

### 3.1 架构

```
请求 → L1 内存 dict (< 0.1ms)
         ↓ 未命中
       L2 SQLite cache 表 (5~20ms)
         ↓ 未命中
       外部 API (500ms ~ 5s)
         ↓
       写入 L1 + L2
```

### 3.2 TTL 分级

| 常量 | 值 | 用途 |
|------|-----|------|
| `TTL_REALTIME_REFRESH` | 5s | 强制刷新 |
| `TTL_REALTIME` | 30s | 实时行情 |
| `TTL_DAILY` | 300s | 日内聚合行情（盘中仍在变化） |
| `TTL_KLINE` | 3600s | K 线历史（日内不变） |
| `TTL_BOARDS` | 1800s | 板块列表/成分股 |
| `TTL_COMPANY` | 86400s | 公司基本面 |

### 3.3 L1 实现（`stock_utils.py`）

```python
_mem: dict[str, tuple[Any, float]] = {}  # key → (value, expires_at)
_mem_lock = threading.Lock()
```

- `get()`: 先查 L1（< 0.1ms），未命中查 L2 SQLite，回填 L1
- `set()`: 同时写 L1 + L2
- L1 过期时间直接来自 SQLite 的 `expires_at`，保证两层一致

### 3.4 启动预热（`app.py _startup()`）

并发预拉取：
1. 基金 ETF DataFrame
2. 板块列表（行业 + 概念）
3. 自选股实时行情（最多 30 只）

用户首次访问命中缓存，无需等待 AKShare。

### 3.5 预期效果

| 场景 | 优化前 | 优化后 |
|------|--------|--------|
| 板块列表（二次访问） | 1~3s | < 5ms（L1） |
| K 线（5 分钟内再访问） | 1~3s | < 5ms（L1） |
| 板块列表（重启后首次） | 2~5s | < 100ms（L2） |
| 板块页面（首次启动后） | 2~5s | < 100ms（预热） |

## 5. 技术分析模块

> 来源：`16-技术分析功能设计与实现`

### 5.1 三项核心能力

1. **K 线画图工具** — 趋势线、水平线、文字标注、矩形框，与个股+周期绑定持久化
2. **形态自动识别** — 十字星、孕线、锤头、射击之星、多头/空头吞噬、MACD 金叉/死叉/顶背离/底背离
3. **分析笔记** — 与个股关联，支持标签

### 5.2 数据库

**`chart_drawings` 表：**

| 列 | 说明 |
|----|------|
| symbol, market, period | 股票+市场+周期绑定 |
| type | `hline` / `trendline` / `text` / `rect` |
| data | JSON 坐标（价格/时间字符串，非像素） |
| color, label, visible | 样式控制 |

**`stock_notes` 表：**

| 列 | 说明 |
|----|------|
| symbol, market | 股票绑定 |
| title, content, tags | 笔记内容 |
| note_date | 笔记日期 |

### 5.3 API 接口（11 个）

**画线：** GET/POST/PUT/DELETE `/api/analysis/drawings` + DELETE `/api/analysis/drawings/clear`

**笔记：** GET tags / GET notes / POST / PUT / DELETE `/api/analysis/notes`

**形态：** GET `/api/analysis/patterns/{symbol}` → 返回与 LightweightCharts `setMarkers()` 兼容的标记

### 5.4 技术形态规则

**蜡烛图形态（纯 NumPy，无 talib）：**

| 形态 | 规则 |
|------|------|
| 十字星 | `body / (range + ε) < 0.1` |
| 孕线 | 当根实体完全在前根实体内（前根 > 1% 收盘价） |
| 多头吞噬 | 前阴后阳，当根阳线完全覆盖前根阴线 |
| 空头吞噬 | 前阳后阴，当根阴线完全覆盖前根阳线 |
| 锤头 | 下影线 ≥ 2×实体，上影线 < 实体 |
| 射击之星 | 上影线 ≥ 2×实体，下影线 < 实体 |

一根 K 线只标注一种形态（按优先级：十字星 > 孕线 > 吞噬 > 锤头 > 射击之星）。

**MACD 信号：**

| 信号 | 规则 |
|------|------|
| 金叉 | DIF 从下穿 DEA |
| 死叉 | DIF 从上穿 DEA |
| 顶背离 | 价格新高但 DIF 未新高（`_find_peaks`，min_distance=5） |
| 底背离 | 价格新低但 DIF 未新低（`_find_troughs`，min_distance=5） |

### 5.5 Canvas 叠加层机制

**pointer-events 策略：**

| 状态 | canvas.pointerEvents | 效果 |
|------|---------------------|------|
| 无工具 / 选择模式 | `none` | 透传给 LightweightCharts |
| 激活画图工具 | `auto` | 拦截鼠标事件 |

**坐标映射：** 画线以价格/时间字符串存储（非像素），每帧重绘时通过 `priceToCoordinate()` / `timeToCoordinate()` 实时转换。

**命中检测：**

| 类型 | 方法 | 阈值 |
|------|------|------|
| hline | `|mouseY - priceToY(price)|` | 8px |
| trendline | 点到线段欧氏距离 | 8px |
| rect | 矩形四条边逐一检测 | 8px |

### 5.6 前端 JS 函数（44 个）

分为：初始化(2)、搜索加载(4)、图表渲染(3)、Canvas 管理(4)、坐标映射(4)、鼠标事件(5)、命中工具(4)、画线持久化(4)、形态识别(3)、笔记管理(6)、导航跳转(2)、清理(3)。

## 7. 内存泄漏修复

### 7.1 问题

6 处 `ResizeObserver` 创建时未保存引用，无法在视图切换/图表重建时 `disconnect()`：

| # | 位置 |
|---|------|
| L1 | `loadKline()` — MACD 子图 |
| L2 | `loadKline()` — RSI 子图 |
| L3 | `selectFund()` |
| L4 | `switchFundKlinePeriod()` |
| L5 | `selectFutures()` |
| L6 | `switchFuturesKlinePeriod()` |

### 7.2 修复方案

1. 声明全局变量：`_klineMacdObserver`, `_klineRsiObserver`, `_fundResizeObserver`, `_futuresResizeObserver`
2. `_disposeKlineSubCharts()` 销毁时先 disconnect observer 再移除图表
3. `loadKline()` 中 MACD/RSI observer 保存到全局变量
4. 基金/期货图表函数顶部先 disconnect 旧 observer
5. `switchView()` 顶部统一清理所有 observer（最后防线）
6. `switchView()` 附加清理自选股定时器

**关键顺序：** disconnect 必须在创建新 observer **之前**执行。
