## TL;DR（电梯陈述）

**这是一个 ~5500 行 Python 实现的「多市场行情 + 微信博主聚合 + 多视角 AI 投资分析 + 交易感知闭环」混合型平台**，同时提供 Web 应用和 MCP 服务两种形态。差异化亮点是**三视角并行 AI 分析（价值派/成长派/趋势派）**、**A 股板块热力图与成分股**、**完整交易感知闭环（喊单→跟踪→验证→统计）**、**财务规则引擎**和**MCP 一等公民**。当前阶段：**功能完整度高、工程化已完成主要改造（路由拆分、SQLite 持久化、双层缓存、连接池）**——适合作为求职作品集和个人决策工具，仍缺单元测试和用户认证。

## 1. 项目定位

### 1.1 一句话定位

**"一个集多市场行情查询、微信博主文章聚合、多视角 AI 投资分析、A 股板块扫描、交易感知闭环于一体的个人投资信息服务平台"**

- **主体**：Web 应用（FastAPI + WebSocket），面向终端用户提供博主文章分析、股票查询、板块行情、交易记录
- **附加**：MCP Server，供 Claude 等 AI 客户端调用微信文章读取能力
- **核心差异化**：多投资视角（价值派 / 成长派 / 趋势派）的 AI 分析 + 交易感知闭环（从博主推荐到实盘/模拟验证的完整链路）

### 1.2 目标用户画像

| 用户群 | 痛点 | 本项目方案 |
|---|---|---|
| 中文资深散户 | 微信公众号信息散乱、缺少结构化 | 博主聚合 + 自动抓取 + AI 分析 |
| 多市场配置者 | A/H/U/期货/基金分散在 N 个 App | 一界面切换全市场 |
| AI 重度使用者 | 通用 AI 无金融语境 | MCP 工具 + 三视角 Prompt |
| 求职 PM / 开发者 | 需要可展示的金融产品作品集 | 这就是作品集本身 |

## 2. 完整能力清单

### 2.1 数据采集

#### 微信文章采集
- **文件**：`src/scraper.py`（234 行）+ `src/parser.py`（74 行）
- **方案**：Playwright 浏览器自动化 + 反检测（隐藏 webdriver 标志、自定义 UA）
- **可靠性**：3 次重试 + SIGPIPE / 网络超时检测
- **输出字段**：title / author / publish_time / content / cover_url
- **统一存储**：`article_storage.py`（106 行）统一管理 WebSocket 采集和定时调度两条采集路径

#### 博主信息识别
- **文件**：`src/blogger.py`（723 行）
- **方案**：从公众号文章 URL 反向识别博主（biz / user_name / avatar / name）
- **双源降级**：mp_backend API → getmsg API 自动故障转移

#### A 股市场数据
- **文件**：`src/stock_service.py`（1011 行）
- **数据矩阵**：

| 数据类型 | 数据源 | 字段说明 |
|---|---|---|
| 实时行情 | 腾讯股票 API（qt.gtimg.cn）| 16 项：实时价/涨跌幅/成交量/市值/换手率/振幅等 |
| K 线 | AKShare（主）+ 腾讯（备）| 日/周/月线前复权，730 天历史 |
| 公司信息 | AKShare 巨潮 | `stock_profile_cninfo` |
| 财务数据 | AKShare 同花顺 | `stock_financial_abstract_ths`，10+ 指标 |
| 资金流向 | AKShare 东方财富 | `stock_individual_fund_flow` |
| 个股新闻 | AKShare | `stock_news_em` |
| 公告 | 巨潮资讯接口 | `stock_notice_report` |
| 股东信息 | AKShare | - |

#### A 股板块行情（新增）
- **文件**：`src/sector_service.py`（590 行）
- **数据源**：东方财富（主）→ 同花顺（备）→ pywencai（第三备）
- **能力**：行业/概念板块列表、成分股（含 PE/市值/换手率）、板块 K 线（日/周/月，2 年历史）
- **热力图**：按涨跌幅排序，7 色阶 CSS Grid 均匀网格

#### 全球市场（韩 / 日股）
- **文件**：`src/global_stock_service.py`（255 行）
- **数据源**：yfinance（KRX、TSE）
- **精选**：韩股 20 只 + 日股 20 只

#### 港股 / 美股
- **文件**：`src/stock_service.py`
- **精选**：港股 30 只 + 美股 100+ 只龙头
- **接口兼容**：与腾讯 API 统一查询协议

#### 同花顺问财
- **文件**：`src/iwencai_service.py`（200 行）
- **能力**：自然语言条件选股、板块热力图、机构调研、板块成分股

#### 基金 / ETF
- **文件**：`src/market/fund.py`
- **数据源**：AKShare `fund_etf_spot_em`
- **分类**：11 类（科技/医药/消费/新能源/金融/军工/资源/宽基/跨境/债券/商品）

#### 期货
- **文件**：`src/market/futures.py`
- **数据源**：AKShare `futures_display_main_sina`
- **分类**：5 类（金属/农产品/能源化工/金融期货/贵金属）

#### 加密货币
- **文件**：`src/market/crypto.py`
- **数据源**：CoinGecko 免费 API，Top 50 + 搜索

### 2.2 数据处理与分析

#### K 线处理
- 日 → 周 / 月聚合
- MA 自定义周期 1-240
- 前复权处理
- 技术指标：RSI / MACD / KDJ / Bollinger（`services/indicators.py`）

#### 财务指标高亮（规则引擎）
- **配置**：`src/financial_rules.json`
- **规则**：净利润同比 / 资产负债率 / ROE / 毛利率 / EPS / 净利润
- **颜色**：red（异常）/ green（优秀）/ orange（警示）

#### AI 投资分析（核心差异化）
- **文件**：`src/analyzer.py`（317 行）+ `src/agents/personas.py`
- **三大投资视角**：

| 视角 | 关注点 | Prompt 关键词 |
|---|---|---|
| **价值派**（buffett） | 护城河、ROE、自由现金流、估值 | PE / PB / 股息率 |
| **成长派**（growth_lynch） | 营收/利润增速、PEG、行业渗透率 | 产品迭代、市场空间 |
| **趋势派**（trend_follower） | 板块轮动、量价配合、强势股梯队 | 技术信号 |

- **并行分析**：多视角同时运行
- **AI 接入**：OpenAI / Claude 兼容 API
- **降级**：无 AI 时自动文本汇总

### 2.3 交易感知闭环（新增）

从博主推荐到实盘验证的完整链路：

| 阶段 | 功能 | 路由 / 文件 |
|---|---|---|
| 推荐追踪 | 博主喊单记录（symbol + 博主 + 时间） | `routers/verify.py` |
| 模拟交易 | 模拟买卖 + 持仓管理 + 回测 | `routers/sim.py` |
| 实盘记录 | 真实交易流水记录 | `routers/verify.py` |
| 交易日记 | 投资心得、操作复盘 | `routers/journal.py` |
| 胜率统计 | 博主推荐准确率、盈亏比分析 | `routers/stats.py` |

### 2.4 服务形态

#### Web 应用（FastAPI）
- **框架**：FastAPI / Uvicorn / Pydantic
- **前端**：纯 HTML 单文件（`templates/index.html`，5541 行）
- **路由**：12 个独立路由模块（`routers/`）
- **图表**：LightweightCharts.js（K 线、热力图）
- **实时**：WebSocket + JSON 协议

#### MCP 服务器
- **文件**：`src/server.py`（92 行）
- **框架**：fastmcp
- **暴露工具**：`read_weixin_article(url: str)`

#### 定时调度（新增）
- **文件**：`src/scheduler.py`（184 行）
- **能力**：定时采集博主文章，支持 latest_n / since_date / period 三种模式

### 2.5 缓存与性能

| 维度 | 实现 |
|---|---|
| L1 缓存 | 进程内 dict（`_mem`），命中无磁盘 I/O，< 0.1ms |
| L2 缓存 | SQLite `cache` 表，持久化，重启不丢失，5~20ms |
| TTL 等级 | REALTIME_REFRESH 5s / REALTIME 30s / DAILY 300s / KLINE 3600s / BOARDS 1800s / COMPANY 86400s |
| 启动预热 | 服务启动时后台并发预拉取基金/板块/自选股行情 |
| HTTP 连接池 | `http_client.py`：全局 `requests.Session`，`trust_env=False`，绕过 VPN 代理 |
| 并发限流 | SemaphorePool（最多 3 并发博主获取） |

## 3. 技术架构

### 3.1 后端架构（当前）

```
前端 SPA (templates/index.html，5541 行)
  → FastAPI app.py (176 行，纯组装)
    → routers/ (12 个路由模块)
      → 业务服务层 (stock_service / sector_service / blogger / analyzer 等)
        → SQLite database.py (WAL 模式，单连接，11 张表)
        → L1 内存缓存 + L2 SQLite 缓存 (stock_utils.py)
        → 外部 API (AKShare / 腾讯 / yfinance / CoinGecko / pywencai)
```

### 3.2 后端栈

| 类别 | 技术 |
|---|---|
| Web 框架 | FastAPI / Uvicorn / Pydantic |
| 异步 | asyncio + Semaphore + asyncio.to_thread |
| 爬虫 | Playwright 1.40+ / BeautifulSoup4 / requests |
| 金融数据 | AKShare / yfinance / pywencai / pandas |
| 数据库 | SQLite（WAL 模式，单连接，11 表） |
| AI | httpx + OpenAI 兼容 API |
| 邮件 | smtplib / email.mime / markdown |
| HTTP 客户端 | requests.Session（全局连接池） |

### 3.3 前端栈

| 维度 | 实现 |
|---|---|
| 构建 | 无打包，纯 HTML + 内联 CSS + 内联 JS |
| 文件 | 单文件 `index.html`（5541 行） |
| 图表 | LightweightCharts.js |
| 设计系统 | CSS 变量 tokens |
| 实时通信 | 原生 WebSocket + JSON |

### 3.4 部署

- 启动：`python app.py`（在 `wexin-read-mcp-main/src/` 下）
- MCP：独立进程 `python server.py`
- 存储：SQLite `data.db`（主存储）+ `bloggers.json`（博主列表）+ `user_config.json`（配置）
- 容器化：未提供 Dockerfile

### 3.5 安全设计

- 邮箱密码 / API Key / 微信 Cookie **永不持久化**
- 仅环境变量读取（`config.py`）
- API 响应自动脱敏
- WebSocket URL 白名单（安全加固）
