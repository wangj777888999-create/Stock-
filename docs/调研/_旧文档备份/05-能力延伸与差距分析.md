## TL;DR（核心判断）

1. **不要试图全面追平 FinceptTerminal**——它是 880 commits / 50+ 屏 / C++ + Python 双语言 / 37 Agent 的体量，正面追赶毫无机会，也不应该是个人项目的目标。
2. **真正的差距集中在 4 个维度**：①投资组合 / 回测 / Watchlist；②AI Agent 数量与深度；③量化能力（QuantLib / Qlib / 回测）；④工程规范（测试 / CI / 文档）。
3. **真正的差异化优势集中在 3 个维度**：①中国市场本地化深度；②微信博主聚合 + 三视角 AI 分析的"内容侧 PMF"；③轻量 Web + MCP 服务化（vs Fincept 的桌面+1GB）。
4. **求职作品集策略**：不要做"中国版 Fincept"——做**「AI-first 的中国散户投研工作台」**。讲清楚定位、讲清楚差异、讲清楚取舍——这比堆功能更重要。

## 2. 差距分类总结

### 2.1 必须追赶（Critical Gap）⭐⭐⭐

求职作品集和"金融 PM 工具"基础合格线：

1. **Watchlist / 投资组合 / 简单回测** —— 任何金融工具的"心智底线"
2. **基础技术指标（RSI / MACD / KDJ / BOLL）** —— K 线模块没这些就是不完整
3. **测试覆盖（≥30%）+ CI** —— 工程严肃度信号
4. **README + ARCHITECTURE.md** —— 让人能看懂项目
5. **`.gitignore` + 依赖锁版本 + 敏感信息审计** —— 安全合规底线

### 2.2 应该追赶（Should-Have）⭐⭐

提升专业度和差异化：

1. **MCP 工具集扩展**（暴露 5-10 个核心工具：行情 / 财务 / 资金 / 三视角分析等）
2. **AI Agent 扩到 5-7 个**（保留 3 派 + 加 Munger / Klarman + Bridgewater 全天候）
3. **龙虎榜 / 北向资金 / 两融**（A 股核心需求）
4. **DCF 简化版 + 风险指标（VaR / Sharpe / 最大回撤）**
5. **数据导出（CSV / Excel）+ 数据持久化（SQLite）**
6. **OpenAPI 文档 + Dockerfile**
7. **响应式 / Mobile 适配**
8. **板块热力图视觉化**
9. **API Key 认证**

### 2.3 不必追赶（Not-Now）

正面打不动，也不该是个人项目目标：

- C++ 桌面端 + Qt UI
- DataHub 自研总线
- QuantLib 18 模块 / Microsoft Qlib / RDAgent
- 19 个地缘政治 Agent / Maritime / Vision Quant
- HFT / 强化学习交易
- 16+ broker 集成
- 节点编辑器 / 代码编辑器内嵌
- Relationship Map / M&A 模块

### 2.4 反向超越（Beat Them）

本项目可以**比 Fincept 做得更深**的方向：

1. ⭐⭐⭐ **中国市场本地化**：龙虎榜、北向、两融、问财、巨潮、节假日
2. ⭐⭐⭐ **微信博主聚合 + AI 分析**（绝对独有）
3. ⭐⭐ **MCP-first 架构**（Fincept 也有 MCP，但所有能力锁桌面 GUI；本项目可作"Claude 用的金融工具集"）
4. ⭐⭐ **轻量 Web 形态**（链接即用 vs 1GB 桌面）
5. ⭐ **财务规则引擎**（配置驱动）
6. ⭐ **更宽松的开源协议**（MIT / Apache 2.0）

## 4. 求职作品集策略

### 4.1 一句话定位（强烈建议固定）

**"AI-first 的中国散户投研工作台 —— 多市场行情 + 微信博主聚合 + 多视角 AI 分析 + MCP 协议接入"**

避免说"中国版 Bloomberg / 中国版 Fincept"——这种对标会被追问"那你比它好在哪"。直接讲**自己的定位**，不和巨头对标。

### 4.2 核心叙事三件套

#### 叙事 1：内容侧的 PMF
**"中国散户的最大信息源是微信公众号——但这个信息源是非结构化的、碎片化的、无法被 AI 直接消费的。我做了一个能把微信博主文章自动抓取、结构化、用三种投资哲学（价值/成长/趋势）并行 AI 分析的工具。"**

**支撑**：scraper / blogger / personas / analyzer 四个模块。

**为什么打动 PM 招聘官**：你能识别中国市场独有的信息源 + 用结构化方式让 AI 消费它——这是产品判断，不只是技术实现。

#### 叙事 2：架构侧的取舍
**"我没做桌面端，因为安装摩擦大；我用 MCP 而不是自研插件协议，因为生态在那里；我没做量化框架，因为那是工具的工具——我做的是给散户和 AI 用的工作台。"**

**支撑**：FastAPI + WebSocket + MCP server + 5 Tab Web UI。

**为什么打动 PM 招聘官**：你能讲取舍背后的判断——"我没做什么、为什么不做"比"我做了什么"更稀缺。

#### 叙事 3：差异化的护城河
**"和海外开源金融终端比，我有中国本地化优势（问财、巨潮、龙虎榜、北向）；和国内同类工具比，我有 AI-first + MCP 优势——任何 Claude / Cursor 都能调用我的工具。"**

**支撑**：能力对比矩阵 + Demo 视频 + MCP 工具列表。

**为什么打动 PM 招聘官**：你能定位自己在市场中的精确位置——这是 PM 的基本功。

### 4.3 应避免的常见错误

1. ❌ 在简历上写"对标 Bloomberg / FinceptTerminal"——会被追问差距并陷入被动
2. ❌ 堆砌技术栈名词（FastAPI / Playwright / WebSocket / MCP / AKShare 全列出来）——重点应在产品判断
3. ❌ 自报夸大数据（"日均处理 10,000 篇文章"）——一问就崩
4. ❌ 讲"未来会做 X"而不是"已经做了 X"——求职作品集只看已交付
5. ❌ 不准备能跑的 Demo——招聘官 30 秒决定要不要继续
6. ❌ 不写 README / ARCHITECTURE——招聘官打开就关掉

### 4.4 应主动突出的卖点

1. ✅ **"我做了 PM 该做的取舍"**——不做什么，比做了什么重要
2. ✅ **"我把不可能 AI 消费的信息源（微信）变成了 AI 工具"**——产品视角的护城河
3. ✅ **"我用 MCP 协议提前一年布局了 AI Agent 时代的金融工具生态"**（FinceptTerminal 的 Programmatic API 在 2026 Q3 才发布）
4. ✅ **"5000 行 Python + 5 Tab UI + MCP，覆盖 7 个市场 + 3 种 AI 视角"**——具体数字 + 边界
5. ✅ **"测试覆盖率 ≥ 30%、Docker 一键部署、CI/CD 全流程"**——工程严肃度
