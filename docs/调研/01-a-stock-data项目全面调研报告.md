## 一、项目概述

### 1.1 定位

a-stock-data 是一个面向中国 A 股市场的**全栈数据工具包**，以 Claude Code Skill（`SKILL.md`）的形式发布。将分散在 8 个公开数据源的 A 股原始数据封装为 AI 编程助手可直接调用的 21 个数据端点。核心卖点：**零 API Key（7/8 源免费）、六层架构、开箱即用**。

### 1.2 目标用户

AI 编程助手用户（Claude Code / Codex / OpenClaw），通过自然语言指令触发数据获取，无需记忆 API 参数。

### 1.3 版本演进

|------|------|------|
| v1.0 | 2026-04 | 内部版，5 层架构，13 个端点 |

## 二、数据体系

### 2.1 数据源全景

项目整合了 8 个公开数据源，按优先级排列：

| 优先级 | 数据源 | 协议 | 鉴权 | 封 IP 风险 | 用途 |
|--------|--------|------|------|-----------|------|
| 1 | **mootdx** | TCP 7709 | 无 | 极低 | K 线、盘口、逐笔、财务、F10 |
| 2 | **腾讯财经** | HTTP | 无 | 低 | PE/PB/市值/换手率/涨跌停价 |
| 3 | **akshare** | HTTP（聚合） | 无 | 中（东财源） | 研报/新闻/公告/龙虎榜/解禁/行业 |
| 4 | **iwencai** | HTTP POST | Bearer+X-Claw | 低 | 自然语言选股/研报搜索 |
| 5 | **东财 PDF** | HTTP | Referer 头 | 低 | 研报 PDF 下载 |
| 6 | **同花顺热点** | HTTP | 无 | 极低 | 强势股筛选 + 题材归因 |
| 7 | **同花顺北向** | HTTP | 无 | 极低 | 沪深股通分钟级资金流 |
| 8 | **百度股市通** | HTTP | Referer 头 | 极低 | 概念板块/资金流向 |

#### Layer 3：信号层（V2.1 大幅扩展，6 个端点）

| 端点 | 数据源 | 响应速度 |

| 百度概念板块 | 行业/概念/地域三维归属 + 涨跌 | HTTP |
| 百度资金流向 | 主力/散户/超大单分钟级 + 20 日历史 | HTTP |

| 端点 | 来源 |
|------|------|

### 2.3 缓存机制

**仅北向资金有缓存**，其余端点无任何缓存：

- 缓存路径：`~/.tradingagents/cache/northbound_daily.csv`
- 策略：每次调用 `hsgt_realtime()` 后自动保存当日收盘数据
- 格式：CSV，列 `date, hgt, sgt`（亿元累计）
- 背景：2024 年 8 月东财停供北向数据后，只能本地积累

**缺失的缓存**：实时行情、K 线、研报列表、行业对比等均无缓存，每次调用都重新请求。

### 2.4 数据格式与解析

| 数据源 | 响应格式 | 解析方式 |
|--------|---------|---------|
| 腾讯财经 | GBK 编码，`~` 分隔 88 字段字符串 | 按索引位置解析（非键值对） |
| 百度资金流（实时） | 自定义分号+逗号分隔格式 | 手动字符串切割 |
| 百度资金流（历史） | 标准 JSON | JSON 解析 |
| 百度概念板块 | JSON（`ResultCode` 类型不一致） | `str()` 强制统一 |
| 东财研报 | JSON（`data` 数组 + `TotalPage`） | 标准 JSON + 手动分页 |
| 同花顺热点 | JSON（`errocode`/`errormsg` 模式） | 标准 JSON |
| mootdx | TCP 二进制协议 | mootdx 库内部解码 |

## 三、架构设计

### 3.1 整体形态

```
a-stock-data/
├── .github/FUNDING.yml    # GitHub 赞助配置
├── assets/                # 图片资源（赞赏码）
├── CHANGELOG.md           # 变更日志
├── LICENSE                # Apache-2.0
├── README.md              # 中英双语说明文档
└── SKILL.md               # 核心文件（~2000 行，Python 嵌入 Markdown）
```

**核心特点**：单文件交付。整个项目是 1 个 SKILL.md 文件，结构化 Markdown + 内嵌 Python 代码块。无独立的 `.py` 文件，无法直接 `python run`。

### 3.2 SKILL.md 内部结构

```
SKILL.md
├── 元数据头（YAML 风格：name/description/version/author）
├── 架构图（ASCII 六层树状图）
├── 激活关键词列表
├── 前置依赖表（mootdx>=0.10, akshare>=1.10, pandas, stockstats）
├── 股票代码前缀规则 + 归一化逻辑
├── Layer 1 函数（mootdx TCP + 腾讯 HTTP）
├── Layer 2 函数（东财研报 + iwencai）
├── Layer 3 函数（信号层 6 个端点）
├── Layer 4 函数（新闻 3 个端点）
├── Layer 5 函数（基础数据 3 个端点）
├── Layer 6 函数（公告 2 个端点）
├── 估值公式（forward_pe / pe_digestion / calc_peg）
├── 投资框架（壁垒→增速→PE 消化→PEG 校验）
├── 4 套调研工作流（单票/批量/主题/新标的）
├── 数据源优先级表
├── FAQ（17 个 Q&A）
└── 安装说明
```

### 3.3 关键设计决策

**决策 1：mootdx TCP 作为行情主通道**

- 不走 HTTP，直接 TCP 二进制协议连接通达信服务器（端口 7709）
- 无 HTTP 开销、无限流、高波动期不会被节流
- 腾讯 HTTP 仅作为估值指标（PE/PB/市值）的补充，不承担 K 线主通道

**决策 2：akshare 作为聚合层**

- 大部分东财/同花顺/cninfo 端点通过 akshare 间接调用
- akshare 内部处理 cookie 管理、反爬 Header、响应解析
- 当 akshare 某接口有 bug（如龙虎榜解析）时，直接绕过调用底层 API

**决策 3：无重试、无熔断**

- 所有调用失败即返回，无自动重试逻辑
- FAQ 中建议用户手动加 `time.sleep()` 防反爬
- 无超时设置（部分 HTTP 调用可能无限等待）

**决策 4：无缓存（除北向）**

- 设计者认为 Skill 场景下用户只查一次，缓存收益低
- 但批量分析场景（50 只股票）会重复请求相同数据

### 3.4 函数签名规范

项目中的函数签名风格混杂：

```python

# 无类型注解（多数）

```

大部分函数缺少参数和返回值的类型注解。

## 四、信息展示与交互设计

### 4.1 交互模式

a-stock-data **没有前端界面**，完全依赖 AI 编程助手的自然语言交互：

| 交互层 | 实现 |
|--------|------|
| 用户输入 | 自然语言指令（中文） |
| 激活机制 | 关键词匹配触发 Skill |
| 输出格式 | 结构化 Markdown（表格/列表/摘要） |
| 可视化 | 无（纯文本） |

### 4.2 使用示例（自然语言触发）

| 场景 | 用户指令 | 调用端点 |
|------|---------|---------|
| 单票估值 | "帮我估一下 688017，给我 PE/PEG/消化时间" | `full_valuation()` |
| 题材归因 | "今天哪些股票走强，主要是什么题材" | `ths_hot_reason()` |
| 研报搜索 | "人形机器人丝杠和减速器最近的研报" | `iwencai_search()` |
| 北向资金 | "今天北向资金流入流出怎么样" | `hsgt_realtime()` |
| 概念板块 | "688017 属于哪些概念板块" | `baidu_concept_blocks()` |
| 资金流向 | "000858 今天主力资金流入还是流出" | `baidu_fund_flow_realtime()` |
| 龙虎榜 | "002475 最近上过龙虎榜吗，哪些营业部在买" | `dragon_tiger_board()` |
| 解禁预警 | "这只股票未来 3 个月有没有限售解禁" | `lockup_expiry()` |
| 行业轮动 | "今天哪些行业涨幅最大" | `industry_comparison()` |

### 4.3 内置投研流程（4 套编排模板）

| 流程 | 步骤 | 耗时 |
|------|------|------|
| **单票估值** | 实时行情 → 一致预期 EPS → 前向 PE → PE 消化年限 → PEG 校验 | 30s |
| **批量对比** | 多只股票横向估值对比（PE/PEG/增速） | 1min |
| **主题研报** | iwencai 多关键词搜索 → 东财 PDF 交叉验证 | 2min |
| **新标的调研** | 覆盖度 → 估值 → 概念板块 → 资金流向 → 龙虎榜 → 解禁预警 | 1min |

### 4.4 估值框架

- **前向 PE** = 当前价格 / 下一年一致预期 EPS
- **PE 消化年限** = 当前 PE 消化到 30x 所需年数（给定 CAGR）
- **PEG** = 前向 PE / (CAGR × 100)。`<1` 便宜，`1~1.5` 合理，`>1.5` 偏贵
- **30x PE 锚点**：A 股成长股的"引力线"

| 方面 | 表现 |
|------|------|

### 5.2 问题清单

| 类别 | 问题 | 严重度 |
|------|------|--------|
| **类型安全** | 大部分函数无类型注解 | 中 |

### 5.3 错误处理模式对比

```
iwencai_search()     → RuntimeError（抛异常）
eastmoney_reports()  → 返回空列表（静默失败）
baidu_fund_flow_*()  → 返回空列表（静默失败）

```

## 九、结论

### 9.1 项目价值判断

a-stock-data 是一个**数据源覆盖度高、上手极快**的 A 股数据工具包。它的核心价值在于：

1. **信号层功能丰富**：龙虎榜、解禁、行业对比、北向资金等是 StockPulse 目前缺失的高价值维度
2. **mootdx TCP 方案成熟**：作为 A 股行情主通道，延迟低、稳定性高
3. **17 个踩坑点文档化**：直接节省排查时间

### 9.2 对 StockPulse 的意义

a-stock-data 与 StockPulse 是**互补关系**而非竞争关系：
- a-stock-data = 纯数据采集工具（AI Skill 形态，无分析/交易/前端）
- StockPulse = 完整投研系统（采集→分析→交易→统计，有前端/数据库/AI 引擎）

**最大参考价值**：数据源广度（尤其信号层）和 mootdx TCP 方案。

### 9.3 建议行动

1. **短期**：接入 mootdx 作为 A 股行情主通道，补充限售解禁和行业对比
2. **中期**：接入龙虎榜、北向资金，消除 http_client 全局锁
3. **长期**：研报检索、估值框架集成、Provider 踩坑文档体系化
