# 调研总览

> 更新时间：2026-05-12
> 本文档汇总 `docs/调研/` 下所有调研报告的核心结论，供快速决策参考。

## 一句话结论

**这个项目的差异化定位已经清晰：AI-first 中国散户投研工作台。不需要对标 Bloomberg，专注「微信博主 + 多视角 AI + 交易验证闭环」三个核心壁垒即可。**

---

## 各报告核心结论

### 竞品对比（01、02）
- **DSA、FinceptTerminal** 等竞品主打数据广度和机构级分析，缺乏中文博主信息源和交易闭环验证
- 本项目的差异化在于**微信信息源 + AI 多视角 + 个人交易感知**，这三点竞品都没有
- 不需要追赶 Bloomberg 级数据深度，那是工程量和成本上的死路

### 能力差距分析（05）
已完成（✅）vs 仍缺（❌）：

| 类别 | 已实现 | 仍缺 |
|---|---|---|
| 数据 | A/H/U/KR/JP + 期货 + 基金 + 加密 | 北向资金、龙虎榜 |
| 分析 | 三视角 AI + 财务规则引擎 + 技术指标 | 策略回测引擎、选股筛选器 |
| 交互 | 板块热力图、K线、博主管理 | 数据导出、多股对比 |
| 工程 | 路由拆分、SQLite、双层缓存 | 单元测试、用户认证 |

**优先追加**：北向资金/龙虎榜（A股核心需求）、数据导出（CSV）

**可以缓**：地缘政治分析、高频交易、期权

### LLM 知识沉淀调研（03）
- 调研了 LLM 长期记忆方案（向量库 / 图谱 / 摘要链）
- 结论：当前阶段暂不需要引入，等 V2 博主分析功能成熟后再考虑知识积累层

### Tiger 均值回归系统调研（04）
- 调研了 Tiger 交易系统的回测框架和均值回归策略
- 结论：`backtests` 表已预留，下一步可实现策略回测引擎（配合 V2 设计文档）

### 技术发现与研究（06）
关键技术坑和解法（已沉淀为代码，此处备查）：

| 问题 | 解法 |
|---|---|
| 微信 MP API 两套接口差异 | mp_backend → getmsg 双源降级 |
| AKShare 被系统代理拦截 | `patch_requests()` + 同时 patch Session 类 |
| 东方财富成分股 ProxyError | 三层降级：东方财富 → 同花顺 → pywencai |
| 股票搜索慢（8.4s） | SQLite 缓存命中后降至 0.005s（1690x） |
| Playwright 反检测 | 隐藏 webdriver + 自定义 UA |
| pywencai 列名含日期后缀 | startswith 匹配重命名 |

---

## 对后续开发的指导

1. **不要铺摊子**：当前功能完整度已经够高，深度优于广度
2. **近期可追加**：北向资金、龙虎榜（AKShare 有现成接口）
3. **V2 方向**：博主评分排行榜（M2-M5）> AI 分析升级 > 通知推送
4. **不追的**：期权、地缘政治、HFT、QuantLib
