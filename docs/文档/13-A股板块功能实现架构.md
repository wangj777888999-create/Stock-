## 一、功能范围

在 StockPulse 中新增 A 股板块页面，包含：

- 行业板块 / 概念板块列表，可按涨跌幅、成交额、涨家数排序
- 涨跌幅热力方块图（颜色深浅 = 涨跌幅大小，方块面积 = 成交额）
- 点击板块进入详情：板块 K 线图（日/周/月）+ 成分股列表
- 点击成分股跳转已有个股详情页
- 双源降级：东方财富（主）→ 同花顺（兜底）；K 线为单源（东方财富）

## 三、后端架构

### 3.1 sector_service.py

#### 整体结构

```
sector_service.py
├── 导入：从 stock_utils 复用 cache / TTL_DAILY / TTL_REALTIME / _clean
├── 导入：从 http_client 复用 patch_requests（代理绕过）
├── PERIOD_MAP：period 英文 → AKShare 中文映射
├── 数据源层（同步函数，供 asyncio.to_thread 包裹）
│   ├── _fetch_boards_eastmoney(board_type)
│   ├── _fetch_boards_ths(board_type)
│   ├── _fetch_stocks_eastmoney(board_name, board_type)
│   ├── _fetch_stocks_ths(board_name, board_type)
│   └── _fetch_kline_eastmoney(board_name, board_type, period)
├── 规范化层
│   ├── _normalize_boards(df) → list[dict]
│   ├── _normalize_stocks(df) → list[dict]
│   └── _normalize_kline(df) → list[dict]
└── SectorService（异步主服务类）
    ├── get_boards(board_type, sort_by, ascending, limit)
    ├── get_board_stocks(board_name, board_type)
    ├── get_board_kline(board_name, board_type, period, count)
    └── search(keyword, board_type)
```

#### 关键设计决策

**复用现有工具，不重复实现**

| 需求 | 使用方式 |
|------|----------|
| NaN 清洗 | `from stock_utils import _clean`（不重写） |
| 代理绕过 | `with patch_requests(): ...`（不手动操作环境变量） |
| TTL 缓存 | `cache.get/set` + `TTL_DAILY`(300s) / `TTL_REALTIME`(30s) |

**缓存分层：原始数据缓存与排序分离**

缓存 key 仅含 `board_type`，不含 `sort_by` / `limit`，避免相同数据因排序参数不同被多次缓存，也避免触发重复的 AKShare 调用：

```python
# 正确做法
raw_ck = f"sector:boards:raw:{board_type}"
raw_data = cache.get(raw_ck)
if raw_data is None:
    raw_data = await _fetch_and_normalize(board_type)
    cache.set(raw_ck, raw_data, TTL_DAILY)

# 在内存中排序 + 截取（不缓存排序结果）
data = sorted(raw_data, key=..., reverse=...)[:limit]
```

**`board_type="all"` 时并发拉取**

```python
# 在 SectorService.get_boards 内，board_type == "all" 时
industry_df, concept_df = await asyncio.gather(
    asyncio.to_thread(_fetch_boards_eastmoney, "industry"),
    asyncio.to_thread(_fetch_boards_eastmoney, "concept"),
)
```

避免串行两次 AKShare 调用（每次约 3–8 秒）导致首屏延迟翻倍。

**K 线 period 映射**

AKShare 接口要求中文 period 值，前端统一使用英文，service 层做映射：

```python
PERIOD_MAP = {
    "daily":   "日k",
    "weekly":  "周k",
    "monthly": "月k",
}
```

**标准输出 Schema**

所有数据源（东方财富/同花顺）的原始列名在规范化层统一映射为以下英文字段：

```python
# 板块列表单条记录
{
    "name": str,
    "type": str,              # "industry" | "concept"
    "change_pct": float,      # 涨跌幅 (%)
    "turnover": float,        # 成交额 (元)
    "up_count": int,
    "down_count": int,
    "lead_stock_name": str,   # 领涨股名称（显示用）
    "lead_stock_symbol": str, # 领涨股代码（前端跳转个股详情用）
}

# 成分股单条记录
{
    "symbol": str,
    "name": str,
    "price": float,
    "change_pct": float,
    "volume": float,          # 成交量（手）
    "turnover": float,        # 成交额（元）
}

# K 线单条记录
{
    "date": str,              # "YYYY-MM-DD"
    "open": float,
    "close": float,
    "high": float,
    "low": float,
    "volume": float,
    "turnover": float,
    "change_pct": float,
}
```

`lead_stock_name` + `lead_stock_symbol` 拆分的原因：东方财富领涨股字段可能仅含名称，需在规范化层尝试提取代码；前端的"点击领涨股跳转个股详情"依赖 `lead_stock_symbol`，缺少时禁用跳转。

#### 缓存 TTL 策略

| 数据类型 | key 格式 | TTL | 说明 |
|----------|----------|-----|------|
| 板块列表原始数据 | `sector:boards:raw:{board_type}` | 300s | 日线档，日内变化慢 |
| 板块成分股 | `sector:stocks:{board_type}:{board_name}` | 30s | 实时档，盘中变动快 |
| 板块 K 线 | `sector:kline:{board_type}:{board_name}:{period}` | 300s | 日线档，日内不变 |

### 3.2 routers/sector.py

```
/api/sector/boards
  GET  ?board_type=all&sort_by=change_pct&ascending=false&limit=100
  → 板块列表，支持排序和数量限制

/api/sector/board/{board_type}/{board_name}
  GET
  → 板块成分股（board_name 为 URL 编码的中文板块名）

/api/sector/board_kline/{board_type}/{board_name}
  GET  ?period=daily&count=120
  → 板块 K 线（单源：东方财富；period: daily|weekly|monthly）

/api/sector/search
  GET  ?keyword=半导体&board_type=all
  → 关键词搜索板块名称（在已缓存数据中过滤）
```

所有参数均带 FastAPI `Query` 校验（枚举值限制、数值范围限制），非法参数返回 422。

## 四、前端架构

### 4.1 页面结构（view-sector）

```
sidebar 按钮（data-view="sector"）
  └── 触发 switchView('sector') → loadSectorBoards()

view-sector
├── Tab 切换：行业板块 / 概念板块 / 全部
│     └── switchSectorTab(type) → loadSectorBoards()
├── 热力方块图（sectorHeatmapCard）
│     └── renderSectorHeatmap(boards)
├── 板块列表表格（sectorListCard）
│     └── renderSectorListTable(boards)
└── 板块详情面板（sectorDetailArea，默认隐藏）
      ├── 返回按钮 → hideSectorDetail()
      ├── 板块 K 线（sectorKlineChart，LightweightCharts）
      │     └── loadSectorKline(type, name, period)
      └── 成分股列表（sectorStocksBody）
            └── loadSectorBoardStocks(type, name)
                  └── 点击行 → openStockFromSector(symbol)
```

### 4.2 全局状态变量

```javascript
let _sectorCurrentType = 'industry';      // 当前 Tab 类型
let _sectorData = [];                      // 当前板块列表数据
let _sectorCurrentBoardName = '';         // 当前详情板块名（供 K 线周期切换使用）
let _sectorCurrentBoardType = 'industry'; // 当前详情板块类型
let _sectorKlineChart = null;             // LightweightCharts 实例
let _sectorKlineResizeObserver = null;    // ResizeObserver 实例
```

`_sectorCurrentBoardName` / `_sectorCurrentBoardType` 在 `loadSectorBoardDetail()` 中赋值，`switchSectorKlinePeriod()` 直接读取这两个变量，**不解析 DOM 文字**（避免标题格式变动导致静默失效）。

### 4.3 JS 函数职责表

| 函数 | 触发时机 | 职责 |
|------|----------|------|
| `loadSectorBoards()` | Tab 切换 / 刷新 / 首次进入视图 | 请求 `/api/sector/boards`，更新热力图和列表 |
| `switchSectorTab(type, el)` | Tab 按钮点击 | 更新 `_sectorCurrentType`，隐藏详情，调 `loadSectorBoards()` |
| `renderSectorHeatmap(boards)` | `loadSectorBoards` 完成后 | 渲染热力方块图（颜色/大小/点击） |
| `renderSectorListTable(boards)` | `loadSectorBoards` 完成后 | 渲染板块列表表格（含领涨股跳转） |
| `loadSectorBoardDetail(type, name)` | 点击方块或表格行 | 赋值状态变量，并发加载 K 线 + 成分股，显示详情面板 |
| `hideSectorDetail()` | 返回按钮 | 隐藏详情面板，显示热力图和列表，销毁 K 线图实例 |
| `loadSectorBoardStocks(type, name)` | `loadSectorBoardDetail` 内 | 请求成分股数据，渲染表格 |
| `loadSectorKline(type, name, period)` | `loadSectorBoardDetail` + 周期切换 | 销毁旧图实例，请求 K 线，创建新 LightweightCharts 实例 |
| `switchSectorKlinePeriod(period, el)` | 日K/周K/月K 按钮 | 读取状态变量，调 `loadSectorKline()` |
| `openStockFromSector(symbol)` | 点击成分股 / 领涨股 | 切换到 stock 视图，填入代码，触发搜索 |

### 4.4 热力方块图渲染规则

- **颜色**：红涨绿跌，透明度随涨跌幅绝对值线性增强（最大 ±10% 时最深）
- **方块面积**：按成交额占最大值的比例缩放，最小 60px，最大 140px（`baseSize = 60 + ratio * 80`）
- **点击**：调用 `loadSectorBoardDetail(type, name)`

### 4.5 `formatTurnover` 冲突检查

添加前先在 `index.html` 中 grep 确认是否已有同名函数：

```bash
grep -n "function formatTurnover" wexin-read-mcp-main/src/templates/index.html
```

若已存在则直接复用，不新增；若不存在则添加：

```javascript
function formatTurnover(val) {
    if (val == null) return '-';
    if (val >= 1e12) return (val / 1e12).toFixed(2) + '万亿';
    if (val >= 1e8)  return (val / 1e8).toFixed(2) + '亿';
    if (val >= 1e4)  return (val / 1e4).toFixed(2) + '万';
    return val.toFixed(0);
}
```

## 五、数据流全链路

```
用户点击"板块"导航
  → switchView('sector')
  → loadSectorBoards()
  → GET /api/sector/boards?board_type=industry&sort_by=change_pct&limit=100
      → routers/sector.py → get_sector_service().get_boards()
          → cache hit → 直接返回
          → cache miss → _fetch_boards_eastmoney (asyncio.gather 并发拉行业+概念)
                       ↓ 失败
                       → _fetch_boards_ths (同花顺兜底)
          → _normalize_boards() 规范化为统一 schema
          → 内存排序 + 截取 limit 条
          → cache.set(raw_ck, raw_data, TTL_DAILY=300s)
          → 返回 {"success": true, "data": [...], "total": N}
  → renderSectorHeatmap() + renderSectorListTable()

用户点击板块方块
  → loadSectorBoardDetail(type, name)
      ├── _sectorCurrentBoardName = name
      ├── _sectorCurrentBoardType = type
      └── Promise.all([
            loadSectorBoardStocks(type, name)
              → GET /api/sector/board/{type}/{name}
              → SectorService.get_board_stocks() → 双源降级 → TTL 30s
            loadSectorKline(type, name, 'daily')
              → GET /api/sector/board_kline/{type}/{name}?period=daily
              → SectorService.get_board_kline() → 东方财富单源 → TTL 300s
          ])

用户点击成分股
  → openStockFromSector(symbol)
  → switchView('stock') → 填入 symbol → searchStock()
```

## 七、已知限制

| 限制 | 说明 |
|------|------|
| K 线无兜底数据源 | AKShare 同花顺无对应 K 线接口，失败时返回友好错误信息 |
| `lead_stock_symbol` 可能为 null | 东方财富领涨股字段不含代码时，列表中领涨股名称不可点击 |
| 前端为过渡实现 | 代码位于单体 `index.html`，待重构时迁移 |
| AKShare 调用耗时 | 首次无缓存时约 3–8 秒，后续 300s 内命中缓存 |
