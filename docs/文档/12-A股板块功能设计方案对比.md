# A 股板块功能设计方案对比

> 撰写日期：2026-05-11
> 状态：**已决策 — 采用方案 B**

## 需求概述

在 StockPulse 中新增 A 股板块功能，支持：
- 浏览**行业板块**和**概念板块**列表
- 可视化展示（词云/热力方块图），按涨跌幅显眼展示
- 板块列表展示：涨跌幅、成交额、领涨股/领跌股
- 点击板块进入成分股列表：基础行情（名称/代码/最新价/涨跌幅）、成交量/成交额
- 点击板块内股票跳转到已有个股详情页
- 板块 K 线图（日 K/周 K/月 K）
- 双源降级：东方财富（主）+ 同花顺（兜底）

## 数据源调研

### AKShare 可用函数

| 函数 | 数据源 | 用途 |
|------|--------|------|
| `ak.stock_board_concept_name_em()` | 东方财富 | 概念板块列表（含涨跌幅、成交额、涨跌家数、领涨股） |
| `ak.stock_board_industry_name_em()` | 东方财富 | 行业板块列表（同上字段） |
| `ak.stock_board_concept_cons_em(symbol=...)` | 东方财富 | 概念板块成分股（含最新价、涨跌幅、成交量、成交额） |
| `ak.stock_board_industry_cons_em(symbol=...)` | 东方财富 | 行业板块成分股（同上字段） |
| `ak.stock_board_concept_hist_em(symbol=..., period=..., adjust=...)` | 东方财富 | 概念板块 K 线历史 |
| `ak.stock_board_industry_hist_em(symbol=..., period=..., adjust=...)` | 东方财富 | 行业板块 K 线历史 |
| `ak.stock_board_industry_name_ths()` | 同花顺 | 行业板块列表（已有集成） |
| `ak.stock_board_industry_cons_ths(symbol=...)` | 同花顺 | 行业板块成分股 |
| `ak.stock_board_concept_name_ths()` | 同花顺 | 概念板块列表 |
| `ak.stock_board_concept_cons_ths(symbol=...)` | 同花顺 | 概念板块成分股 |

### 现有集成

- `iwencai_service.py`：已使用 `ak.stock_board_industry_name_ths` 获取行业板块列表，`get_sector_stocks()` 使用 pywencai 自然语言查询成分股（非结构化）
- `routers/iwencai.py`：已有 `GET /api/iwencai/sectors` 和 `GET /api/iwencai/sector/{name}`
- 东方财富接口数据更结构化，字段更全（含涨跌幅、成交额、换手率等）
- 同花顺接口已有部分集成，适合做兜底

### 腾讯 API

腾讯公开 API 不支持板块/行业/概念数据查询，仅支持个股行情和 K 线。不适用于本功能。

## 方案 A：扩展 MarketProvider 模式

### 概述

在 `market/` 目录下新建 `a_share_sector.py`，实现 `MarketProvider` 抽象类。注册后自动获得 `/api/market/sector/*` 路由。前端新增 `view-sector` 页面。

### 后端改动

**新建文件：`src/market/a_share_sector.py`**

```python
class AShareSectorProvider(MarketProvider):
    name = "sector"
    label = "A股板块"

    async def get_boards(self, board_type="all"):
        """行业+概念板块列表，双源降级"""
        # 优先东方财富 stock_board_industry_name_em + stock_board_concept_name_em
        # 降级同花顺 stock_board_industry_name_ths + stock_board_concept_name_ths
        # 返回 [{"name": ..., "code": ..., "change_pct": ..., "turnover": ...,
        #         "up_count": ..., "down_count": ..., "lead_stock": ...}]

    async def get_board_stocks(self, board_name, board_type="industry"):
        """板块成分股"""
        # 优先东方财富 *_cons_em，降级同花顺 *_cons_ths

    async def get_spot(self):
        """板块实时行情快照（用于热力图）"""
        # 返回板块涨跌幅排序数据

    async def search(self, keyword):
        """按关键词搜索板块"""
```

### 前端改动

在 `index.html` 中新增 `view-sector` 视图，包含：
- 行业/概念 Tab 切换
- 板块热力方块图（`_renderSectorHeatmap()`）
- 板块列表表格（涨跌幅、成交额、领涨股）
- 成分股详情面板 + 板块 K 线图表

### 优点

- 严格遵循现有 MarketProvider 架构
- 自动注册路由，无需手动维护路由注册
- 与基金、期货、币圈页面风格一致

### 缺点

- `MarketProvider` 接口只有 4 个抽象方法（`get_boards`、`get_board_stocks`、`get_spot`、`search`），不支持：
  - 按涨跌幅/成交额排序
  - 区分行业/概念类型
  - 板块 K 线数据
- 需要扩展基类接口或在子类中新增方法，打破接口一致性
- `get_spot()` 语义与板块不太匹配（板块不是"实时报价"概念）

## 方案 B：扩展 iwencai_service + 专用路由（推荐 ✅ 已选）

### 概述

新建 `sector_service.py` 封装板块业务逻辑和双源降级，新建 `routers/sector.py` 专用路由。复用项目已有的 `patch_requests`、`cache`、`call_with_fallback` 等模式。

### 后端改动

**新建文件：`src/sector_service.py`**

```python
class SectorService:
    """A 股板块服务：行业板块 + 概念板块，双源降级"""

    async def get_boards(self, board_type="all", sort_by="change_pct", ascending=False, limit=100):
        """
        获取板块列表
        board_type: "industry" | "concept" | "all"
        sort_by: "change_pct" | "turnover" | "up_count"
        limit: 返回条数上限，默认 100
        返回: {"success": True, "data": [...], "total": N}
        """

    async def get_board_stocks(self, board_name, board_type="industry"):
        """获取板块成分股，优先东方财富，降级同花顺"""

    async def get_board_kline(self, board_name, board_type="industry", period="daily", count=120):
        """
        获取板块 K 线（东方财富 *_hist_em，无兜底）
        period 接受前端传入的英文值，内部通过 PERIOD_MAP 转换为 AKShare 所需中文值
        """

    async def search(self, keyword, board_type="all"):
        """搜索板块"""

    # --- 内部降级方法 ---
    async def _fetch_boards_eastmoney(self, board_type):
        """东方财富数据源"""
        # ak.stock_board_industry_name_em() / ak.stock_board_concept_name_em()

    async def _fetch_boards_ths(self, board_type):
        """同花顺数据源（兜底）"""
        # ak.stock_board_industry_name_ths() / ak.stock_board_concept_name_ths()

    async def _fetch_stocks_eastmoney(self, board_name, board_type):
        """东方财富成分股"""
        # ak.stock_board_industry_cons_em() / ak.stock_board_concept_cons_em()

    async def _fetch_stocks_ths(self, board_name, board_type):
        """同花顺成分股（兜底）"""
        # ak.stock_board_industry_cons_ths() / ak.stock_board_concept_cons_ths()

    async def _fetch_kline_eastmoney(self, board_name, board_type, period, count):
        """东方财富板块 K 线（单源，无兜底）"""
        # ak.stock_board_industry_hist_em() / ak.stock_board_concept_hist_em()
        # period 映射：PERIOD_MAP = {"daily": "日k", "weekly": "周k", "monthly": "月k"}
```

复用的现有模式：
- `patch_requests()` 来自 `http_client.py`，AKShare 调用代理绕过
- `asyncio.to_thread()` 包裹同步调用
- `cache.get()` / `cache.set()` 来自 `stock_utils.py`，TTL 缓存
- `_clean()` 数据清洗（NaN → None）
- `call_with_fallback()` 双源降级（来自 `stock_service.py`）

#### 单例注册（state.py）

项目所有服务（`stock_service`、`scraper`、`blogger_mgr` 等）均作为单例挂在 `state.py` 上，`SectorService` 需遵循同样模式，否则每次请求都实例化会导致内部缓存失效：

```python
# state.py 中添加
sector_service: SectorService = None

def get_state() -> AppState:
    ...

# app.py lifespan 中初始化
state.sector_service = SectorService()
```

Router 中通过 `get_state().sector_service` 获取实例，与其他路由保持一致。

#### 依赖声明

`requirements.txt` 需补充 `akshare` 依赖（当前 `stock_service.py` 已在 import 但未声明）。

#### 标准输出 Schema（数据规范化）

东方财富和同花顺返回的列名不同，需在 `SectorService` 内部统一映射到以下英文字段，再对外返回，避免降级切换时前端拿到不同字段名：

```python
# 板块列表单条记录
{
    "name": str,              # 板块名称
    "type": str,              # "industry" | "concept"
    "change_pct": float,      # 涨跌幅 (%)
    "turnover": float,        # 成交额 (元)
    "up_count": int,          # 上涨家数
    "down_count": int,        # 下跌家数
    "lead_stock_name": str,   # 领涨股名称（用于显示）
    "lead_stock_symbol": str, # 领涨股代码（用于前端跳转个股详情，如 "000001"）
}

# 成分股单条记录
{
    "symbol": str,        # 股票代码
    "name": str,          # 股票名称
    "price": float,       # 最新价
    "change_pct": float,  # 涨跌幅 (%)
    "volume": float,      # 成交量 (手)
    "turnover": float,    # 成交额 (元)
}
```

#### K 线 period 映射

AKShare 的 `stock_board_industry_hist_em` / `stock_board_concept_hist_em` 接受中文 period 值，Service 层需在内部做一次转换，前端统一使用英文值：

```python
PERIOD_MAP = {
    "daily":   "日k",
    "weekly":  "周k",
    "monthly": "月k",
}

# 使用示例
ak_period = PERIOD_MAP.get(period, "日k")
```

#### 并发拉取（board_type="all"）

当 `board_type="all"` 时，行业板块和概念板块需同时获取。应用 `asyncio.gather()` 并发请求，避免串行造成延迟翻倍（AKShare 单次调用约 3–8 秒）：

```python
async def _fetch_boards_eastmoney(self, board_type):
    if board_type == "all":
        industry, concept = await asyncio.gather(
            asyncio.to_thread(ak.stock_board_industry_name_em),
            asyncio.to_thread(ak.stock_board_concept_name_em),
        )
        # 合并后统一规范化
    elif board_type == "industry":
        ...
```

#### 缓存 TTL 策略

| 数据类型 | TTL | 说明 |
|----------|-----|------|
| 板块列表（行业/概念） | 300s | 日内变化较慢，对应现有日线档 |
| 板块成分股 | 30s | 盘中实时性要求较高 |
| 板块 K 线 | 300s | 历史数据，日内不变 |

### 已有端点迁移计划

- `GET /api/iwencai/sectors` — 同花顺行业板块列表

**注意**：`iwencai_service.py` 中存在两类不同的同花顺调用，迁移时需区分对待：

| 现有逻辑 | 调用方式 | 迁移策略 |
|----------|----------|----------|
| 行业板块列表 (`ak.stock_board_industry_name_ths`) | AKShare 结构化接口 | 整合进 `_fetch_boards_ths()` 作为降级数据源 |
| 板块成分股 (`get_sector_stocks`) | pywencai 自然语言查询 | **保留在 `iwencai_service.py`**，作为独立的"自然语言板块查询"功能，不迁移 |

新端点上线后的处理：
1. AKShare THS 板块列表逻辑整合进 `SectorService._fetch_boards_ths()` / `_fetch_stocks_ths()` 作为降级数据源
2. `GET /api/iwencai/sectors` 在 `routers/iwencai.py` 中标记 deprecated（保留接口、响应头加 `Deprecation` 字段），待前端完成切换后删除
3. `GET /api/iwencai/sector/{name}`（pywencai 自然语言查询）保留，不废弃，功能独立
4. 前端板块列表/成分股统一切换到 `/api/sector/*` 新端点

## 方案 C：独立服务 + 专用路由 + Provider 适配器

### 概述

同方案 B 建立独立的 `sector_service.py` + `routers/sector.py`，额外实现一个轻量 `MarketProvider` 适配器注册到 market 系统，使板块功能既可以通过 `/api/sector/*` 专用路由访问，也能通过 `/api/market/sector/*` 统一入口访问。

### 后端改动

**新建文件：`src/sector_service.py`**（同方案 B）

**新建文件：`src/routers/sector.py`**（同方案 B，丰富 API）

**新建文件：`src/market/a_share_sector.py`**

```python
class AShareSectorProvider(MarketProvider):
    """轻量适配器，将 SectorService 适配到 MarketProvider 接口"""
    name = "sector"
    label = "A股板块"

    def __init__(self):
        self._svc = SectorService()

    async def get_boards(self):
        """适配：调用 SectorService.get_boards()"""

    async def get_board_stocks(self, board_name):
        """适配：调用 SectorService.get_board_stocks()"""

    async def get_spot(self):
        """适配：返回板块涨幅榜"""

    async def search(self, keyword):
        """适配：调用 SectorService.search()"""
```

**修改文件：`src/market/__init__.py`** + **`src/app.py`**

同时注册 Provider 和路由。

### 优点

- 既有专用 API（`/api/sector/*`）又有统一 market 入口（`/api/market/sector/*`）
- 职责分离：Service 负责业务逻辑，Router 负责 HTTP 层，Provider 负责 market 适配

### 缺点

- 多了一层适配器，增加了维护成本
- 两套路由指向同一逻辑，可能造成困惑
- MarketProvider 接口限制仍在（4 个方法不够用）
- 复杂度最高，收益有限

## 对比总结

| 维度 | 方案 A (MarketProvider) | 方案 B (专用路由) ✅ | 方案 C (双注册) |
|------|------------------------|---------------------|-----------------|
| 新建文件数 | 1 (provider) | 2 (service + router) | 3 (service + router + provider) |
| 修改文件数 | 2 (__init__.py + market.py) | 2 (app.py + index.html) | 3 (__init__.py + app.py + index.html) |
| API 灵活度 | 低（受限于接口） | 高 | 高 |
| 板块 K 线 | 需扩展接口 | 直接支持（单源） | 直接支持（单源） |
| 排序/筛选 | 需扩展接口 | 直接支持 | 直接支持 |
| 架构一致性 | 高（纯 Provider） | 中（独立路由） | 中（双重入口） |
| 复杂度 | 低 | 低 | 高 |
| 复用现有代码 | 少 | 多 | 多 |
| 推荐度 | ★★☆ | ★★★ | ★★☆ |

### 决策：采用方案 B

**决策日期**：2026-05-11

**核心理由：**
1. 复用最多现有代码和模式（`patch_requests`、`cache`、`call_with_fallback`）
2. 专用路由不受 `MarketProvider` 4 方法接口限制，K 线/排序/分页/搜索均可直接支持
3. 前端参考基金视图的热力图模式已有成熟实现
4. 改动范围最小，只需新建 2 个文件 + 修改 2 个文件
5. `iwencai_service.py` 的 THS 板块逻辑可整合进降级链路，无需重复开发

**实现时需注意的事项（已补充进方案 B 章节）：**
- 单例注册：`SectorService` 挂载到 `state.py`，避免缓存失效
- 数据规范化：东方财富/同花顺字段映射到统一 schema（英文字段名）
- `lead_stock` 拆分：schema 需同时返回 `lead_stock_name`（显示）和 `lead_stock_symbol`（跳转）
- K 线 period 映射：`"daily"→"日k"`、`"weekly"→"周k"`、`"monthly"→"月k"`（AKShare 要求中文值）
- K 线无兜底：明确单源，失败时返回友好错误而非空数据
- Service/Router 签名对齐：`SectorService.get_boards()` 需含 `limit` 参数
- 并发拉取：`board_type="all"` 使用 `asyncio.gather()` 避免串行延迟
- 分页：`/api/sector/boards` 默认 `limit=100`，防止 500+ 概念板块渲染卡顿
- 旧端点迁移：`/api/iwencai/sectors` 标记 deprecated 并删除；`/api/iwencai/sector/{name}`（pywencai）保留不迁移
- 前端过渡：板块视图加进 `index.html` 为过渡实现，待前端重构后独立为组件
- 缓存 TTL：板块列表 300s，成分股 30s，K 线 300s
